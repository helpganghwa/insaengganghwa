## 1. 강화 — 시간 곡선 & effective rate

### 1.1 시간 곡선 — "도달 누적 시간" 기준 (강화 레벨만)

- 베이스/카탈로그/슬롯 **전부 무관** — 모든 장비 동일 (GDD §3.2)
- 앵커는 **1회 시도 시간이 아니라 +0 → 해당 레벨 도달까지의 누적 평균 시간** (full-wait 전략 = 매 시도 완료까지 대기해 baseRate 최대치, 평균 재시도 포함)

**누적 도달 앵커 (사용자 확정):**

| 도달 레벨 | +0 → 누적 평균 시간 |
|-----------|---------------------|
| +30 | ≈ 24시간 (1일) |
| +50 | ≈ 3일 (72h) |
| +99 | ≈ 2주 (336h) |

- idle 친화: 누적이 길어도 떠나 있는 동안 진행. 보석 단축으로 압축, +100~는 시간 plateau + 제물

**1회 시도 소요 d(L) — 추천 (시뮬레이션으로 정밀화)**

누적 ≈ `Σ_{k≤L} d(k) × 평균 재시도수(≈ 1/baseRate(k), §1.3)`. baseRate가 낮아질수록 재시도↑ → 누적 증가의 대부분은 **고레벨 저확률 재시도**에서 발생. 단일 시도는 짧게 유지:

| L | d(L) 추천 | L | d(L) 추천 |
|---|-----------|---|-----------|
| +1~5 | 3s·6s·12s·25s·50s | +35 | 75m |
| +6~9 | 2m·4m·8m·15m | +40~50 | 80~85m |
| +10 | 20m | +60~75 | 85~90m |
| +20 | 50m | +90~99 | 90m |
| +30 | 70m | +100~ | ≈90m + 1제물/시도 |

- 단일 시도는 +20 이후 대략 **1~1.5h로 완만 유지** — 누적 증가는 주로 낮은 baseRate × 재시도에서 발생 (의도)
- **+100 제물 규칙**: 대상과 같은 카탈로그 아이템이면 1개 = 제물 1 (강화·초월 레벨 **무관**, **+0도 가능**). 초월 제물 규칙과 동일 (GDD §3.2/§3.3)

> 구현: `d(L)` 표 + `baseRate(L)` 표(§1.3)를 `lib/game/balance.ts` 박제. `scripts/simulate-enhance`(Vitest balance)가 누적 앵커(+30→24h, +50→3일, +99→2주, full-wait)를 충족하도록 `d(L)` 정밀화 — 본 문서와 1:1.

### 1.2 시간 비례 effective rate

```
p_eff(L, elapsed, total) = baseRate(L) × clamp(elapsed / total, 0, 1)
```

- `elapsed` = 큐 등록 후 서버 시계 경과 (보석 단축분 포함), `total` = 등록 시점 산정된 총 소요 시간
- 큐 **완료(elapsed ≥ total)** 시 `p_eff = baseRate(L)` (공시 최대치)
- 일찍 시도할수록 선형으로 낮은 확률 → 도박 (GDD §3.2)
- `baseRate(L)` 정의 = §1.3 (다음 블록)
- RNG는 **시도 시점 서버**에서 `crypto` 기반 결정 (CLAUDE §3.1)

### 1.3 성공률 baseRate(L) & 하락 (추천값)

`baseRate(L)` = **끝까지 기다렸을 때의 공시 성공률**. 실제 = `baseRate(L) × clamp(경과/총, 0, 1)` (§1.2).

| 레벨 구간 | 실패 결과 | baseRate (추천) |
|-----------|-----------|-----------------|
| +0 → +9 | — (항상 성공) | **100%** |
| +10 → +20 | 유지 (안전) | 100% → 85% (레벨당 ≈ −1.4%) |
| +21 → +30 | 유지 (안전) | 85% → 70% |
| +31 → +40 | 유지 (안전) | 70% → 58% |
| +41 → +51 | 유지 (안전) | 58% → 50% |
| +52 → +60 | **−1 하락** | 48% → 38% |
| +61 → +75 | **−1 하락** | 38% → 25% |
| +76 → +90 | **−1 하락** | 25% → 15% |
| +91 → +99 | **−1 하락** | 15% → 10% |
| +100 ~ | **−1 하락** | 10% 고정 (하한) + 제물 1/시도 |

**하락 규칙**
- 하락은 **+52 이상에서 실패 시에만** 발생, 폭 = **−1 레벨** (단순·예측가능, 파괴 없음 — GDD §3.2)
- 하락 하한 = **+51** (실패로 +52→+51 회귀, +51 이하는 안전 구간이라 더 안 떨어짐)
- 자동 재등록: 성공 시 다음 레벨 자동 큐잉 / 실패(유지·하락) 시 중단 + 결과 모달 (GDD §3.2)

**설계 의도**
- 안전 구간(+0~+51)은 실패해도 시간만 손해 → baseRate를 비교적 높게(50%+) 유지해 "기다리면 거의 됨" 체감
- 하락 구간(+52~)은 −1 손실 + 낮은 baseRate + effective rate 도박성 → 보석 단축/제물 투자 동기
- 구현: 레벨→bp(만분율) **단조 감소 표**를 `lib/game/balance.ts`에 박제, 본 표와 1:1 (게임산업법 §33 공시 일치)

## 2. 초월 — 제물 수 & 전투력 % 배수 (추천값)

초월: 대상과 같은 카탈로그 아이템(강화/초월 레벨 무관, +0 가능) 제물 소모 → `transcend_level +1`. **최대 10**. 즉시·무RNG, 제물 수만 충족하면 100% (GDD §3.3).

### 2.1 레벨업 제물 수 (추천: 선형 1→10)

| 초월 단계 | 해당 단계 제물 수 | 누적 |
|-----------|------------------|------|
| 0→1 | 1 | 1 |
| 1→2 | 2 | 3 |
| 2→3 | 3 | 6 |
| 3→4 | 4 | 10 |
| 4→5 | 5 | 15 |
| 5→6 | 6 | 21 |
| 6→7 | 7 | 28 |
| 7→8 | 8 | 36 |
| 8→9 | 9 | 45 |
| 9→10 | 10 | 55 |

- 한 아이템 풀 초월(10)에 **같은 아이템 누적 55개** 제물 필요 (대상 본체는 별도)
- 보급 상자는 슬롯 내 균등 랜덤이라 특정 아이템 중복 수집 = 장기 엔드게임 목표 (의도된 병목, GDD §3.4)

### 2.2 전투력 % 배수 (추천: 가속 곡선)

`개별 장비 전투력 = (강화 기반 전투력) × (1 + transcendBonus[T])`

| T | transcendBonus | T | transcendBonus |
|---|----------------|---|----------------|
| 0 | +0% | 6 | +45% |
| 1 | +5% | 7 | +56% |
| 2 | +11% | 8 | +68% |
| 3 | +18% | 9 | +82% |
| 4 | +26% | 10 | **+100% (×2.0)** |
| 5 | +35% | | |

- 고초월일수록 단계 이득 **가속** (GDD §3.3) — 초월 10 = 그 장비의 강화 전투력을 2배로
- 구현: `transcendFodder[T]`, `transcendBonusBp[T]` 표를 `lib/game/balance.ts`에 박제, 본 표와 1:1

## 3. 전투력 (추천값)

### 3.1 강화 레벨 → 전투력 P(L)

모든 카탈로그 아이템 **동일 곡선** (베이스 차등 없음, GDD §3.1).

추천식: `P(L) = round(10 × (1 + L)^1.5)` — 초선형(지수 1.5)이라 고강화 투자가 전투력으로 확실히 보상.

| L | P(L) | L | P(L) |
|---|------|---|------|
| 0 | 10 | 50 | 3,643 |
| 5 | 147 | 51 | 3,750 |
| 10 | 365 | 60 | 4,775 |
| 20 | 962 | 70 | 5,982 |
| 30 | 1,726 | 80 | 7,290 |
| 40 | 2,608 | 99 | 10,000 |

(계수 10·지수 1.5는 추천 시작값 — 곡선 가팔기는 튜닝 가능)

### 3.2 개별 / 총 전투력

- **개별 장비 전투력** = `P(L) × (1 + transcendBonus[T])` (§2.2)
- **총 전투력** = `(착용 3장비 전투력 합) × (1 + 도감강화합 × 0.005)`
  - 도감강화합 = 도감에 기록된 **카탈로그 아이템별 최고 강화 레벨의 합** (Σ, 전 카탈로그)
- 예: 무기 L50·T2 = 3,643 × 1.11 ≈ 4,044 / 도감강화합 200이면 총 전투력 = (3슬롯 합) × (1 + 200×0.005) = ×2.0

### 3.3 랭킹 지표 (GDD §5 — 시즌제 없음, 상시 누적)

- **최고 강화** = 보유 전 장비 중 단일 최고 강화 레벨 (max L)
- **합산 강화** = 도감강화합 (Σ 카탈로그별 최고 L) — 총 전투력 도감 항과 동일 소스
- **전투력** = 위 총 전투력

> `P(L)` 표·`0.005` 계수는 `lib/game/balance.ts`에 박제, 본 문서와 1:1.

## 4. 보급 (추천값)

### 4.1 카탈로그 (슬롯별, 가변)

- 카탈로그 아이템은 **슬롯(무기/방어구/장신구)별 분류**, 종수는 **고정 아님 — 지속 추가 예정** (GDD §10)
- 모든 아이템 성능 동일 (외관·도감·초월 동일성 판정용, GDD §3.1)
- 초기 예시: 무기 검/도끼/완드/활/스태프… · 방어구 갑옷/부츠/장갑/투구/방패… · 장신구 아뮬렛/귀걸이/팔찌/목걸이/반지…

### 4.2 박스 개봉 확률

- 보급 상자는 **슬롯 일치** — 무기 상자→무기만, 방어구 상자→방어구만, 장신구 상자→장신구만
- 슬롯 내 **완전 균등**: `P(아이템) = 1 / (해당 슬롯 현재 카탈로그 종수)` — 종수가 늘면 확률 자동 재분배(항상 모두 동일 확률), **천장 없음**
- 획득 아이템 항상 **+0 · 초월0**
- 게임산업법 §33: 박스별 현재 아이템 목록·균등 확률을 게임 내 + `/probability` 공시 (카탈로그 추가 시 갱신·사전 고지)
- **공시 페이지 범위(§33은 확률형 한정)**: `/probability`는 *확률형* 요소만 공시 — §1 강화 성공률 · §2 초월(결정적이나 참고) · §3 전투력 · §4 보급 슬롯 균등 당첨. **레이드·재화/보상은 결정적(추첨 없음)이라 공시 페이지에서 제외**. 레이드·경제 *밸런스 스펙*은 본 문서 §5/§6에 유지(메커닉 운영 중) — 공시 의무 대상이 아닐 뿐.

### 4.3 분해 (확정)

- 장비 분해 시 **보석 고정 지급** (확률 아님) — **10 보석/개**, 강화·초월 레벨 **무관 고정**
- 중복은 보통 초월/+100 제물로 사용 → 분해는 잉여 정리 + 보석 회수 경로
- **개봉 보석 드롭 폐기**: 이전 "개봉 20% 확률 보석 1~3"는 제거(확률형 보상 미운영). 보급 개봉의 확률형 요소는 슬롯 내 균등 당첨뿐.

> 균등 확률 규칙·분해 보석은 `lib/game/balance.ts` 박제, 본 문서와 1:1. 카탈로그 종수는 가변이므로 확률은 "균등(1/현재 슬롯 종수)" **규칙**으로 박제(고정 수치 아님).

## 5. 레이드 (추천값)

> **재화 표기**: 다이아 ≡ 보석 — **단일 프리미엄 재화**(diamond). 강화 단축·레이드 개설/추가공격·IAP 모두 동일 재화 (상세 §6).

### 5.1 개설 / 참여 상수

| 항목 | 값 |
|------|----|
| 개설 비용 | **1,000 다이아** (즉시 차감, 환불 없음) |
| 최대 인원 | 10명 (호스트 포함) |
| 동시 진행 (유저당) | **3개** (호스팅+참여 합산) |
| 일일 진행 한도 (유저당) | **5회/일** (KST 기준) |
| 공격창 | 개설 후 **6시간** (서버 시계) |
| 기본 공격 횟수 | **10회 / 참여자** |
| 추가 공격 | 다이아로 구매 (§5.5) |

### 5.2 페이즈 HP

- phase1 HP = 랜덤 정수 `U(8000, 12000)`
- phase n HP = `phase1 × 1.5^(n-1)` (페이즈마다 ×1.5)
- 죽이는 개념 없음 — 6시간 만료로만 종료, **돌파 페이즈 수 = 보상 규모** (GDD §3.5)

### 5.3 1회 공격 데미지

```
dmg = round( 총전투력 × k × U(0.7, 1.3) × (크리 ? 1.5 : 1) )
```

- `k = 1.0` (추천 시작값 — 시뮬로 조정), 총전투력 = §3.2
- **미스 없음** (항상 명중) · 크리 확률 **5%** · 크리 배수 **×1.5** · 데미지 분산 **±30%** · **데미지 캡 없음**

### 5.4 보상 (전원 동일 — 기여도 무관)

1회 이상 공격한 참여자 **전원 동일** 지급:

- **기본 참가**: 100 다이아
- **페이즈 돌파마다 추첨 1회** (페이즈별 1회 굴림 → 전원 동일 적용):
  - 50% → **100 다이아**
  - 50% → **보급 상자 1개** (무기 / 방어구 / 장신구 각 **1/3**)
- 기여도 가중 추첨 **없음** (GDD §3.5). 정산: 6시간 만료 시 lazy + cron 멱등 (CLAUDE §3.4)

### 5.5 추가 공격 (다이아 구매)

- 기본 10회 소진 후 다이아로 추가 공격 구매 (해당 레이드 한정, 공격창 내)
- n번째 추가 공격 비용 (**점진 증가**): `50 + 10 × (n−1)` → 50 · 60 · 70 · 80 …
- 추가 공격도 §5.3 동일 데미지 규칙

> 모든 수치 `lib/game/balance.ts` 박제, 본 문서와 1:1. `k`·크리·보상·추가공격 곡선은 시뮬/플레이테스트로 조정.

## 6. 경제 (추천값)

### 6.1 재화 — 단일 프리미엄

### 6.2 강화 시간 단축 환산 (확정)
- **1 다이아 = 1분 단축**

### 6.3 공유 보상 (확정)

- 별도 유틸리티 "우편 즉시 수령" 광고 = 적재 우편 1건 즉시 처리 (보상 아님 — GDD §3.7/§3.10)
