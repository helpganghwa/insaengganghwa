## 1. 장비 모델 (재설계) — 모든 것의 토대

인스턴스 더미를 폐기하고 **카탈로그당 단일 레코드**로 수렴.

### 1.1 데이터 모델
`equipment_instances` + `user_codex` → **`user_equipment`** (PK `user_id, catalog_item_id`)
| 컬럼 | 의미 |
|---|---|
| `enhance_level` | 현재 강화 레벨 (카탈로그당 1개) |
| `transcend_level` | 현재 초월 레벨 |
| `transcend_progress` | 다음 초월까지 누적 중복 수 |
| `max_enhance_level` / `max_enhance_reached_at` | 역대 최고(배틀패스·랭킹) |
| `max_transcend_level` / `max_transcend_reached_at` | 역대 최고 |
| `equipped_slot` | 장착 슬롯(외형) / null |
| `first_acquired_at` | 도감 해금 시각 |

### 1.2 규칙
- **박스 열기**(랜덤 카탈로그 1종): 미보유 → 획득(+0/T0). 보유 → `transcend_progress +1` → 임계 도달 시 **자동 초월**.
- **자동 초월**: 선형 — T→T+1에 중복 (T+1)개. 무한 진행. 도달 즉시 자동 발동, `transcend_level +1` 및 `max_transcend_level` 갱신, progress는 다음 목표로 이월.
- **강화**: 순수 시간 + RNG (+ 다이아 1/분 단축). **제물 없음**. 카탈로그당 강화레벨 1개.
- **전투력**: 카탈로그별 레코드 = P(강화) × (1+초월보너스), 보유 카탈로그 합. (best-instance dedup 불필요)

### 1.3 삭제 (전부)
- `equipment_instances` 인스턴스 보관 개념
- 분해 / 일괄분해 (`disenchant`, `DIAMOND_PER_DISENCHANT`, `disenchant_logs`, BulkDisenchantModal, 관련 UI/공시)
- 수동 초월 / 일괄 초월 (`transcend/perform.ts` 수동 경로, BulkTranscendModal)
- +100↑ 강화 제물 규칙
- 효과: 잉여 장비/인벤 클러터 소멸, 모든 중복이 항상 초월 전진(낭비 없음).

### 1.4 초월 수치 (이미 일부 구현됨, 자동초월이 그대로 사용)
- 제물: 선형 `toT` 개, 무한.
- 전투력 보너스: T0~T10 가속곡선(T10=+100%), **T11~ 레벨당 +10%p** 무한.

## 2. 경제 — 다이아 (전면 리밸런싱)

### 2.1 단가 정책
목표 **하루 ₩5,000~10,000**. 소액=비싸게/대량=싸게이되 격차 완만(1.18×). 다이아 ≈ ₩4.5/💎.

### 2.2 IAP 패키지
| 패키지 | 가격 | 💎 | =며칠치 | ₩/하루 |
|---|---|---|---|---|
| starter | ₩1,500 | 300 | 0.21 | ₩7,200 |
| small | ₩6,000 | 1,200 | 0.83 | ₩7,200 |
| medium 🔖인기 | ₩13,000 | 2,800 | 1.94 | ₩6,690 |
| large | ₩28,000 | 6,400 | 4.44 | ₩6,300 |
| mega 🔖최대혜택 | ₩68,000 | 16,000 | 11.1 | ₩6,120 |

\+ **첫 결제 2배**(1회 한정, 신규 전환).
> 세트상품(다이아+보급)·정밀 패키지 가격은 별도 리밸런싱 예정.

### 2.3 무료 다이아 faucet (passive, 보장 17,000/28일)
| 출처 | 신규 | 28일 |
|---|---|---|
| 일일 보급(매일) | 300💎 | 8,400 |
| 출석 평일 순환(×12) | 300💎 | 3,600 |
| 출석 14일 마일스톤 | 2,000💎 | 2,000 |
| 출석 28일 마일스톤 | 3,000💎 | 3,000 |
> 보급권/박스는 별도 자원(2.6). passive를 줄이고 **경쟁(대난투)으로 버는 모델**.

### 2.4 대난투(melee) 보상 — 핵심 경쟁 faucet
| 등수 | 💎 | 📦 |
|---|---|---|
| 1등 | 2,000 | 30 |

> 평균 ≈ 95💎/인/일(floor가 평균 지배), 1등 2,000은 글로벌 1명/일 희소상. 총 faucet ≈ 19,600/28일(평균).

### 2.5 기타 faucet
- 초대(전환): 300💎 + 보급상자 3개 (유지)
- 레이드 페이즈 돌파: 보급상자 1개/페이즈 (다이아 0)
- **분해: 삭제** (다이아 faucet 아님)

## 3. 배틀패스 (신규 기능)

### 3.1 개념
**성장 패스**(시즌·만료 없음). 종류: **강화 패스 / 초월 패스**. **계정 최고 도달(lifetime max)** 기준 — 하락·소모와 무관 단조. 구간별 **독립 결제**, 산 구간만 프리미엄 수령(이미 넘긴 단계는 즉시 소급 일괄). 무료 라인은 전 구간 항상 수령.

### 3.2 구간 & 보상 (단계당 지급)
- 강화 패스(보상=다이아), 구간 +1~100 / +101~200 …, **무료 10×2^c / 프리미엄 50×2^c** 단계당
- 초월 패스(보상=보급상자·무작위슬롯), 구간 T1~10 / T11~20 …, **무료 10×(c+1) / 프리미엄 50×(c+1)** 단계당
- (c = 구간 인덱스 0부터, 무료 = 프리미엄 1/5)

### 3.3 프리미엄 구간 가격 (현금/포트원, no-brainer)
| 구간 | 강화 패스 | 초월 패스 |
|---|---|---|
| c0 | ₩9,900 | ₩9,900 |
| c1 | ₩19,900 | ₩19,900 |
| c2 | ₩39,900 (×2) | ₩29,900 (선형) |
> "안 사면 손해" 정체성 — 보상 가치 대비 ~2.3배. 진행 게이트 + 구간당 1회 + 캡으로 패키지 자기잠식 제한.

### 3.4 최고도달 추적
`user_equipment.max_enhance_level / max_transcend_level`의 계정 MAX = 패스 진행 기준 (`getMaxReached`). 단조 보장.

## 4. 단계별 구현 로드맵 (의존순)

> 각 단계: 작업마다 커밋 / 마이그레이션은 적용(허가됨) / 배포(master push)는 요청 시.
> docs-read-as-fresh: 정식 문서엔 "삭제·변경 이력" 없이 최종 설계인 듯 정리.

### Phase 1 — 장비 모델 (토대, 최대 작업)
1. 스키마 `user_equipment` 신설 + 마이그레이션(`equipment_instances`+`user_codex` 데이터 이관)
2. 박스 열기 재작성 → 획득 / `transcend_progress+1` / 자동초월
3. 자동초월 로직(수동 `perform.ts` 대체) + 최고기록 갱신
4. 강화 재작성 → 제물 제거, 카탈로그당 1레벨
5. 전투력/리더보드 쿼리 → 카탈로그 레코드 기반
6. 삭제: 분해 전부 · 수동/일괄초월 · 일괄분해 · 인스턴스 개념
7. UI: 인벤=카탈로그 그리드, 강화 화면, 초월(자동) 표시, 모달 제거
8. `/probability` 공시: 분해 제거, 초월(자동)·강화(무제물) 갱신 / SCHEMA·GDD·BALANCE 동기화

### Phase 2 — 경제 수치
1. `balance.ts`: IAP 패키지, 일일/출석 다이아, 대난투 보상, sink(아바타·닉변·레이드개설·추가공격), `DIAMOND_PER_DISENCHANT` 제거
2. BALANCE.md §6·§7·§8 + `/probability`(§33) 동기화
3. 밸런스 시뮬/테스트 갱신

### Phase 3 — 배틀패스 (신규)
1. 스키마: 패스 진행/구간 구매/수령 상태
2. `balance.ts`: 구간 보상 곡선 + 가격
3. 진행(최고도달 키잉)·소급 수령·무료/프리미엄 라인·프리미엄 결제(포트원) 로직
4. UI: 패스 화면(강화/초월, 구간, 무료/프리미엄 트랙, 수령)
5. 테스트 + 공시/문서
