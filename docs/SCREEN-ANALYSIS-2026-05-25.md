# 인생강화 화면/페이지 구성 분석 보고서

> 작성일: 2026-05-25
> 분석 대상: 17개 화면(public 2 + game 14 + admin 2) + 글로벌 셸
> 관점: 게임 기획자/디자이너 + 사용자 제안(피드형 홈) 평가

---

## 0. TL;DR

**기술적·시각적 완성도 A−.** 390 컬럼 고정 + 픽셀아트 정체성 + idle 핵심 루프와 정합되는 깔끔한 5탭 구조. **그러나 동선·정보 밀도·소셜 노출에 구조적 약점이 있다.** 사용자가 직관적으로 느낀 "홈에 버튼 나열이 아쉽다"는 정확히 그 약점의 지점이다.

| 영역 | 점수 | 핵심 메시지 |
|------|------|----|
| 글로벌 셸(헤더/바텀네비) | B+ | 알림 dot 정확하나 우편/공지 진입점 부재 |
| 홈(메뉴 허브) | C+ | grow식 런처 패턴 — 매 접속 동일, "다녀왔지만 새로운 게 없다" |
| 강화 (코어) | A | 카드 정보 밀도·결과 연출 모두 우수 |
| 인벤토리/도감/프로필 | B | 깔끔하나 도달까지 1~2 hop |
| 랭킹 | B− | 단독 화면이라 자기 동기와 격리됨 |
| 우편함 | B | 헤더에 빠르고 자연한 진입점이 없음 |
| 보급/레이드/상점 | A− | 자체로는 완성도 높음 |

**사용자 제안 결론**: 방향 정답. 단 **전면 교체가 아닌 하이브리드**(메뉴 그리드 축소 + 활동 피드 추가)를 권장. 자세한 근거는 §4.

## 1. 글로벌 셸 (모든 화면 공통)

### 1.1 잘한 점
- **390 고정 + safe-area** 처리가 모든 화면에 일관 적용 (CLAUDE §5.2 권위 잘 지켜짐)
- **다크 단일 테마** 결정으로 색 일관성 ↑, 픽셀아트 가독성 ↑
- **알림 dot** — 강화 완료 시점 도달 시 BottomNav `⚒️ 강화`에 빨간 점. 서버 시계 정확, 사용자가 "뭔가 끝났다"를 인지 가능
- **`(game) layout`의 lazy daily mail** — 핫패스에 가벼운 INSERT/no-op, 사용자가 매 접속에서 일일 보급을 자동 보장
- **AppHeader 우측 다이아 = 상점 직링** — paywall 동선 짧음

### 1.2 못한 점
- **헤더에 우편함 진입점이 없다** — WIREFRAMES §0에도 "배치 위치 후속 확정"으로 명시되어 있음. 우편함이 비동기 보상 적재처(GDD §3.10)인데도 헤더 아이콘이 없어 사용자가 "내 보상이 어디서 쌓이지?"를 모름
- **헤더에 알림/공지 진입점이 없다** — 운영 공지(점검·이벤트)가 닿을 길이 없음
- **헤더 우측이 "닉네임 · 💎 다이아"만** — WIREFRAMES §0에는 `🏆 내 랭킹(#순위)` 압축 표시 의도가 있었으나 미구현
- **5탭 중 `🎒 인벤토리`가 코어 동선과 약함** — 강화 화면에서 장비 선택이 다 가능(picker 모달)이라 인벤토리 탭의 일상적 유용성 낮음
- **5탭에 우편함·랭킹 없음** — 둘 다 홈 메뉴 경유라 2 hop. 매 접속 빈도 높은 우편함이 특히 불편

### 1.3 권장
- **헤더 우측에 `📬` 우편 미수령 카운트 배지** 추가 — 1 hop으로 단축
- BottomNav를 `홈 / 인벤토리 / 강화 / 레이드 / 프로필` → `홈 / 강화 / 우편함 / 레이드 / 프로필`로 재배치 검토. 인벤토리는 헤더 또는 강화/프로필 내부 진입
- 헤더 좌측 로고 옆에 시스템 모드(점검·이벤트) 상태 라이트 (read_only일 때 안내 띠) — 운영 가시성

## 2. 홈 (메뉴 허브, 2×4 그리드)

### 2.1 잘한 점
- **픽셀아트 카드 8장** = 매우 강한 시각 identity. grow식 런처 패턴은 첫 인상 좋음
- 각 메뉴의 **한 줄 설명 + tint 배경**으로 어떤 화면인지 명확
- 강화/인벤토리/보급/레이드/우편함/도감/프로필/랭킹 — **8개 카드로 게임 전체를 1 hop에 노출**

### 2.2 못한 점
- **매 접속에서 동일한 8개 카드** — "다녀왔지만 뭐 새로운 거 있나?" 신호가 없음. idle 게임의 핵심 KPI(재접속)와 미스매치
- **일일 보급 카드가 없다** — WIREFRAMES §1에 명시된 "오늘의 보급 카드(미수령=수령하기 / 수령완료=카운트다운)"가 코드에 미구현. **이게 가장 큰 누락**. 일일 미션 reminder 없음
- **자기 진행 신호 0** — 강화 진행 중인 큐 수, 마지막 +99 도달, 도감 진척률 등 "내가 어디까지 와있다"는 자기 reflection 없음
- **사회적 자극 0** — 친구·다른 유저가 무엇을 하는지 보이지 않음
- **카드 8개 × 4/3 비율** = 1뷰포트(820pt 기준)에서 거의 모두 보이지만, 정보 밀도가 낮음(8개 = 라벨 16자)
- **운영 메시지 진입점 없음** — 이벤트/패치 노트/점검 안내 노출 자리 없음

### 2.3 영향
- D2~D14 재접속 시 "변화 없음" 체감 → 이탈 가속
- "자랑 → 신규 유입" 핵심 루프의 마지막 단(자랑 답방 노출)이 약함
- 일일 미션·도전 메타가 부재해 "오늘 뭘 해야 하지?" 명확성 0

## 3. 강화 (코어 화면)

### 3.1 잘한 점
- **카드당 정보 밀도가 매우 높고 잘 짜여 있음** — 스프라이트·초월 배지·레벨 전환·성공/유지/하락·시간·즉시 단축 비용·취소까지 한 카드 100px에 다 들어감
- **테두리 진행 게이지** (BorderGauge) — 픽셀 동적 표현으로 idle "차오름" 정서 강화
- **결과 플래시 + 대장장이 로어** — 1.6초 오버레이 + 톤 분기로 정서 변별
- **컨펌 모달**: 완료 전이면 "최대 확률 아닙니다" 경고, 완료면 즉시 강화 — early attempt 도박성 보호
- **3슬롯 × 2 lane = 6 카드** — 한 화면에 모두 표시 가능, 동시 강화 UX 직관적

### 3.2 못한 점
- **빈 lane 카드의 정보 빈약** — 카드 본문이 "+ 강화 등록" 버튼만. 후보 자동 추천(가장 높은 강화 가능한 장비)이 없어 매번 picker 모달 거쳐야 함
- **6개 카드를 세로로 나열** = 스크롤 발생. 슬롯별로 토글 가능한 collapse 또는 viewport 적응적 1×N 레이아웃 검토
- **즉시 단축 비용이 카드 우측 작게** — paywall CTA로 정렬이 약함. 헤비 유저에겐 핵심 buttoning 동선

### 3.3 권장
- 빈 lane에 "추천: +47 룬소드 / 86분 후 완료" 같은 1-tap shortcut
- 즉시 단축 CTA를 더 강조 (시간 게이지 옆 inline `완료 -2h: 💎120`)

## 4. 사용자 제안 평가 — "버튼 나열 X, 랭킹·공지·유저 역사 피드 O"

이 부분은 비중 있게 다룹니다. 핵심 결정이라.

### 4.1 사용자 제안의 정확성

**방향은 정답**입니다. 이유:

| 근거 | 설명 |
|------|------|
| **게임 KPI와 정렬** | "+99 자랑" 컨셉(IDEA.md)의 핵심 루프 마지막 단 = 노출. 자랑 피드가 곧 게임 컨셉의 시각화 |
| **한국 모바일 게임 표준** | 양산형 RPG·idle 거의 모두 메인 화면이 활동/이벤트 피드. 사용자 학습된 mental model에 부합 |
| **재접속 동기 강화** | "매일 새로운 콘텐츠가 있다"는 신호 — idle의 가장 큰 약점인 D7+ 이탈 방어 |
| **사회적 자극** | 다른 유저의 자랑이 내 도전 동기로 — 카톡 외부 친구가 없는 1인 플레이 환경 보강 |
| **운영 메시지 도달** | 1인 운영자가 공지·이벤트를 알릴 자리 신설 |

### 4.2 그러나 — 단점 인지

| 단점 | 영향 | 완화 |
|------|------|------|
| **첫 인상 비주얼 손상** | 픽셀아트 카드 그리드는 현재 identity의 핵심. 텍스트 피드로 완전 교체 시 시각적 매력 ↓ | 하이브리드(카드 4개 + 피드 하단) |
| **빈 상태 처리 어려움** | 출시 직후 DAU 100 미만에서는 "다른 유저 활동" 적어 피드가 빈약 | 시드 데이터·운영자 공지·도전 미션으로 채움 |
| **1인 운영 부담** | 피드 큐레이션·공지 작성·이벤트 운영 추가 부담 | 자동화(자랑 이벤트 자동 적재) + 수동(공지)는 minimal |
| **인프라 신설** | 현재 활동 피드 테이블·쿼리·UI 컴포넌트 모두 없음 | `home_feed` 단일 테이블 + Cache Components로 비용 통제 |
| **390 컬럼 스크롤 깊이 ↑** | 메뉴 그리드는 1뷰포트 / 피드는 무한 스크롤 | 상단 4카드 + 중단 피드 fold 3-5건만 + "더보기" |
| **GDD §2 짧은 체크인 세션 의도와 약간 충돌** | "강화 결과 확인 → 다음 큐 등록 → 보급"의 30초~2분 동선에 피드가 마찰을 더할 수 있음 | 강화 진입은 BottomNav 1-tap 유지 |

### 4.3 권장: 하이브리드 홈 (전면 교체 X)

|---|------|------|------|

### 4.4 활동 피드 데이터 소스 (구현 가능성)

이미 존재하는 데이터로 거의 가능:

| 피드 종류 | 데이터 소스 | 신규 테이블 |
|----------|------------|------------|
| 내 강화 +30/+50/+99 도달 | `enhancement_logs` 필터 | 불요 |
| 내 초월 T1/T10 달성 | 초월 로그(`transcend_logs`) | 불요 |
| 내 레이드 정산 | `raid_rewards` | 불요 |
| 내 도감 신규 해금 | `user_codex` insert | 불요 |
| 내 랭킹 변동 | `leaderboard/queries` 비교 (이전 vs 현재) | 단순 캐시 1개 |
| 운영 공지 | **신규** `announcements` 테이블 (1인 운영용 admin 페이지) | 신규 |
| 타 유저 자랑 (P1+) | `shares` 테이블 join | 불요 |

핵심: **운영 공지 테이블 1개 + 통합 쿼리 뷰 1개**로 출시 가능.

### 4.5 결정 사항 권고

1. **즉시 도입**: 오늘의 보급 카드 + 내 진행 카드 + 헤더 우편 배지 — 1~2일 작업
2. **출시 전 도입**: 운영 공지 테이블 + 자기 활동(강화 도달/초월/도감) 피드 5건 — 3~5일
3. **출시 +1개월**: 타 유저 자랑 피드 + 친구 활동 (P1)
4. **메뉴 그리드는 축소 유지**: 4카드 + 활동 피드 하이브리드. 전면 피드형 교체는 idle 게임의 짧은 세션 의도(GDD §2)와 충돌

## 5. 페이지별 빠른 평가

### 5.1 인벤토리 `/inventory`
**Good**: 슬롯 필터 query string, 카탈로그 join 효율적, lore 미리 적재
**Bad**: 카드 그리드인데 한 화면 노출량 제한, 정렬/필터 옵션 부족(레벨·잠금·장착 기준)
**개선**: 정렬 토글, 일괄 분해 모드, 검색

### 5.2 도감 `/me/codex`
**Good**: 카탈로그 150종 vs 보유 비교, 챔피언 표식
**Bad**: 미획득은 disable 이미지만 보여 "어떻게 얻나" 불명확. 슬롯별 진척률 헤더 부재
**개선**: 슬롯별 N/M 진척 헤더, 미획득에 "무기 보급 상자에서 등장" 안내

### 5.3 프로필 `/me`
**Good**: 장착 3슬롯 + 총 전투력 + 도감 합 + 자랑 진입(BoastLauncher). 핵심 자기 정체성 한 화면
**Bad**: 메뉴 4개(도감·랭킹·설정·확률 공시)가 다 외부 화면 진입. 자기 페이지인데 정작 자기 정보 밀도가 약함
**개선**: 최근 강화 활동 5건, 최고 도달 기록, 자랑 횟수·총 공유 클릭수

### 5.4 랭킹 `/leaderboard`
**Good**: 3 metric 탭(최고/합산/전투력) + 내 순위 노출
**Bad**: 단독 화면이라 진입 비용. 다른 사람 닉네임만 보임, 그 사람 장비 상세 모름
**개선**: 1위~10위 닉네임 → 공개 프로필 링크. 내 순위 변동(↑3) 표시

### 5.5 우편함 `/mail`
**Good**: unread/done 탭 분리, 만료 lazy
**Bad**: 헤더 진입점 없어 도달성 ↓. 푸시 알림 도착해도 우편함 진입까지 2 hop
**개선**: 헤더 배지(§1.3)

### 5.6 보급 `/gacha`
**Good**: 박스별 픽셀아트 카드 + LCP preload
**Bad**: 일일 보급(KST 00:00 충전)이 어디서 오는지 흐릿. 슬롯 카탈로그 종수 표기 없어 균등 확률 신뢰감 약함
**개선**: 카드 하단 "현재 N종 균등" 표기, /probability 인라인 링크

### 5.7 레이드 `/raid`
**Good**: 동시 진행 + 일일 카운트 헤더, 카드 슬롯
**Bad**: 호스트 1000다이아 부담 안내가 빈약(개설 직전에야 노출)
**개선**: 활성 레이드 카드 상단에 "참여 가능한 친구 레이드" 섹션(카톡 외 발견 경로)

### 5.8 상점 `/shop`
**Good**: 본인인증 가드 안내, 5단계 패키지
**Bad**: 결제 백엔드 미연동(준비 중 상태)이 명확히 안 보임. 보너스 비율 한국 표준보다 약함(BUSINESS-ANALYSIS §3.2)
**개선**: BUSINESS-ANALYSIS B0-2 적용

### 5.9 설정 `/me/settings`
**Good**: 푸시 카테고리 토글(2026-05-25 도입), 닉네임 변경 cost, 본인인증 상태
**Bad**: "약관/개인정보처리방침" 비활성(DisabledRow) — 출시 전 필수
**개선**: 약관 페이지 작성, 데이터 다운로드/계정 삭제(GDPR/개인정보보호법 대비)

### 5.10 공개 프로필 `/u/[nickname]`
**Good**: 비로그인 구경 가능 — "가입 마찰 완화" 핵심
**Bad**: 본 보고서 작업 범위 외(별도 검토 권장)

### 5.11 확률 공시 `/probability`
**Good**: 코드와 1:1 (§33 위반 0). 사이클·3분기 모델 동기화됨
**Bad**: 홈에서 진입 경로 없음(프로필에서만)
**개선**: 보급 화면 카드 하단 inline 링크

## 7. 한 줄 결론

> 사용자의 직관(`버튼 나열 → 피드`)은 게임 KPI와 정확히 정렬된 정답. 단 **전면 교체가 아닌 하이브리드** — 메뉴 그리드를 4카드로 축소하고, 그 위/아래에 **오늘의 보급 + 내 진행 + 활동 피드** 3섹션을 더한다. 이러면 첫 인상의 픽셀아트 identity는 보존되면서 "다녀왔는데 새로운 게 있다" 신호가 매일 들어온다. 가장 큰 누락(`오늘의 보급 카드` — WIREFRAMES §1에 이미 명세된 것이 미구현)부터 먼저 채우면 ROI 가장 큼.
