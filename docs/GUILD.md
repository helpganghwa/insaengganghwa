# GUILD — 길드 (협력 성장 · 지역 점령)

> 유저가 길드를 결성해 **함께 길드를 키우고, 월드맵의 지역을 공격·수비해 점령**하는 사회·경쟁 콘텐츠. 강화/초월로 쌓은 개인 전투력이 길드 전투력으로 환원되고, 점령은 서버 결정론 정산으로 판정되므로 실시간 PvP 인프라·치트 위험 없이 1인 운영에 친화적이다. 게임 전 시스템을 묶는 **엔드게임 리텐션 축**.

관련: `GDD.md`(코어 루프) · `BALANCE.md §3`(전투력) · `MELEE.md`(결정론 정산 모델 선례) · `SCHEMA.md`(스키마) · `MAIL.md`(경고·보상 우편) · `PROFILE.md`(Pixellab 파이프라인 — 문양 런타임 생성)

## 0. 설계 원칙 (협상 불가)

- **결정론·비동기 (대난투 모델)**: 실시간 공성 없음. 점령은 정해진 KST 시각에 길드 전투력 스냅샷 → 결정론 정산. 안티치트·1인 운영 친화.
- **등급 없음·공정성 유지**: 기부·과금이 **개인 전투력으로 환산되지 않는다**. 돈으로 점령력/길드 진척을 독식하는 P2W 금지(즉시 정체성 훼손). 길드 수익화는 **외형·집단 편의** 한정.
- **반-스노볼**: 1등 길드 영구 독식 방지. 개인 일일 기부 상한·길드 수용 상한(30)·**영주제(보유 구역당 영주 1명 필수)**로 격차 누적을 구조적으로 억제.
- **idle 저압력 유지**: 미션은 "매일 새로 하라"가 아니라 **하던 행동의 자동 집계**. 기여 의무가 숙제가 되지 않게.
- **출시 시퀀싱**: 길드는 **마지막 콘텐츠**. 인구가 있어야 재밌으므로 정식 오픈 후 **DAU 임계(길드가 빠르게 차는 수준) 확인 뒤 투입**. 콜드스타트 단계에 넣으면 "빈 길드 = 죽은 게임" 신호가 되어 역효과.

> 모든 수치는 **시작 테스트값**이며 `lib/game/balance.ts` 단일 출처로 이관 + `tests/balance/` 시뮬레이션으로 튜닝한다(CLAUDE §3.5).

## 1. 길드 결성 / 가입 / 탈퇴

- **결성 비용**: **10,000💎**. 결성자는 길드장. 개설은 소수(길드당 1명)·가입은 다수 무료라 높은 비용이 전체 진입장벽이 아니며, **즉흥 개설로 인한 빈 길드 양산을 억제**하고 다이아 sink로 기능한다(앵커 1,440💎/일 대비 ≈ 7일치).
  - (선택·TBD) 비-다이아 자격 게이트 병행 검토 — 예: 계정 생성 N일 경과 + 최소 합산 전투력. 진지한 길드장 필터를 다이아 액수 인상 없이 강화.
- **길드명**: 자유 텍스트(예 2~12자), **결성 시 확정·변경 불가**(문양만 재생성 가능 §1.6). **사전 AI 검수 없음** — 부적절명은 **신고 기반 사후 대응**(운영 강제 개명/해산). 중복 허용 여부 TBD.
- **수용 인원**: `15 + (길드레벨 − 1)`, **상한 30**(레벨 16에서 도달). §2 곡선과 맞물림.
- **길드 아이덴티티** → §1.5.
- **탈퇴 후 24시간 재가입 불가**: 용병성 길드 호핑·정찰 어뷰징 방지. 서버 타임스탬프 비교(결정론·구현 단순).
  - 적용 범위: 탈퇴 후 24h간 **다른 길드 가입 불가**.
  - ⚠ 월드맵 막판 갈아타기(정산 직전 승자 길드로 점프)는 24h 잠금만으로 부족 — **점령 보상 자격을 시즌/정산 시점에 묶는 규칙**으로 별도 보강(§5 TBD).
- **해산**: 길드장 **자발 해산** 또는 운영 **강제 해산** 시 — 보유 구역 전부 **중립화**, **세금 풀 소멸**, 멤버 방출.

## 1.5 길드 아이덴티티 (이름 · 문양 · 공지)

- **이름**: §1 — 자유 텍스트, 사전 검수 없음, 신고 기반 사후 대응.
- **문양(엠블럼)**: **3축(모양·색상톤·키워드) 선택 → Pixellab 런타임 생성** → §1.6. 자유 업로드·자유텍스트 없음(고정 어휘라 모더레이션 안전).
  - **BM 연결(§6)**: 재생성(5,000💎)·프리미엄 모양/색상톤·한정 키워드는 비-P2W 외형 매출 후보.
- **공지**: 짧은 자유 텍스트(예 ≤60자). **편집 권한 = 길드장/부길드장만**. **사전 AI 검수 없음** — 멤버 전용 노출 + 신고 기반 사후 대응. idle 특성상 자유 채팅(§7 보류) 대신 v1의 1줄 소통 채널.

> 사전 검수를 두지 않으므로 **이름·공지에 대한 신고 경로와 운영 처리(개명/해산/제재) 동선**은 반드시 마련한다(사후 모더레이션 의존).

## 1.6 문양 — 키워드 조합 생성

- **방식**: **3축 선택 → Pixellab 런타임 생성**(길드마다 고유). 모양 1 · 색상톤 1 · 키워드 1~3 선택 → 조합 프롬프트 구성 → 생성. (전부 고정 어휘 → 모더레이션 안전.)
- **① 모양(1택, 6)**: 라운드 방패 / 카이트 방패(세로) / 문장 배너 / 사각 문장판 / 날개 문장 / 마름모
- **② 색상톤(1택, 7) = `emblem_color`**: 핏빛 적 / 황금 / 심해 청 / 독성 녹 / 마력 보라 / 흑철 / 백은 (유저 선택색이 곧 맵 소유 틴트)
- **③ 키워드(1~3택, 20)**: 용 · 늑대 · 뱀 · 까마귀 · 그리핀 · 교차검 · 해골 · 투구 · 도끼 · 왕관 · 사자 · 눈 · 크리스탈 · 별·달 · 불꽃 · 화산 · 천사 날개 · 슬라임 · 오크 토템 · 신전
- **생성 시점·비용**: 길드 결성 시 **무료 1회 생성(고정)** — 3축 검증 결과 스타일 일관성 충분(유저가 모양·색·키워드로 제어). 이후 **재생성 = 5,000💎**(💎 sink·외형 BM). 마음에 안 들면 재생성으로 커버.
- **길드명 변경 불가**(결성 시 확정). 문양만 재생성 가능.
- **일관성·품질 안전장치(설계)**: 모양·색상톤은 변수, **스타일만 고정 앵커**(픽셀아트 · 오르네이트 테두리 처리 · **중앙 단일 볼드 모티브** · 투명배경) → **소형(맵 노드 16~24px) 가독성**. 키워드가 많아도 "중앙 1개 모티브"로 단순화.
- **용도**: 길드 프로필 · 멤버 목록 · 검색 · **월드맵 구역 소유 표시(작게)** · 점령전 UI. **맵 소유 색** = 유저가 고른 **색상톤(`emblem_color`)** 직접 사용(추출 불필요).
- **기술**: 생성물 저장 `public/sprites/guild/emblems/<guildId>.png`(또는 blob), `guilds.emblem_url`. 서버에서 Pixellab 호출 → **"문양 생성 중" 로딩 + 실패 재시도 + 기본 문양 폴백**(API 장애 대비). 생성비 ~$0.02는 결성 10,000💎 / 재생성 5,000💎로 충분 상회.
- **모더레이션**: 키워드 풀이 고정이라 부적절 생성 위험 낮음(자유텍스트 없음). 생성물 사전검수 없이 **신고 기반 사후 대응**(§7).
- **프롬프트 템플릿(검증됨)**: `generate-image-pixflux`, 128×128, `no_background:true` —
  > `a dark fantasy pixel art guild heraldic emblem, a {모양} crest in a {색상톤} color palette, featuring {키워드들}, a single bold central symmetrical motif, ornate metal border, clean strong silhouette, bold and readable at small sizes, centered, transparent background, no text`
  - 모양→`round shield`/`tall vertical kite shield`/`heraldic banner pennant`/`square heraldic crest plaque`/`winged crest`/`diamond lozenge crest`, 색상톤→`crimson blood-red`/`golden`/`deep ocean blue`/`toxic green`/`arcane purple`/`dark iron black`/`silver white`.

## 2. 길드 성장 (기부 · 미션 → 길드 XP → 레벨)

### 2.1 기부 (일 3회, KST 자정 리셋)

| 기부 | 비용 | 길드 XP | 개인 기여도 |
|---|---|---|---|
| 1회차 | 무료 | 10 | 10 |
| 2회차 | 50💎 | 30 | 30 |
| 3회차 | 100💎 | 70 | 70 |

- 일 최대 **110 길드 XP / 150💎** per 멤버.
- **왜 체증 + 상한**: 고래도 하루 150💎로 동일 상한 → 돈으로 길드 진척 독식 불가(반-스노볼). 무료 1회는 캐주얼의 무압력 데일리 탭.
- **왜 효율 0.6→0.7 XP/💎**: 100💎(70XP)가 50💎(30XP)보다 다이아당 효율이 살짝 높아, **3회차까지 채울 동기** 부여(무료만 찍고 마는 것 방지).
- 경제 정합: 앵커 **1,440💎=하루** 대비 일 기부 150💎 ≈ **10%** — 부담 없는 sink.

### 2.2 미션 (자동 집계형 — 하던 행동이 쌓임)

- **일일**: 강화 시도 · 보급 개봉 · 대난투 참가 등 합계 **~50 길드 XP/일** (세부 항목 TBD).
- **주간**: 누적 목표 **~250 길드 XP/주**.
- 활성 멤버 1인 ≈ 하루 **150~200 길드 XP** 기여(기부+미션 전부 채울 때).

### 2.3 길드 레벨 곡선 (앞 싸게 = 수용 빨리 열림 / 뒤 비싸게 = 엔드게임 sink)

| 구간 | 레벨당 필요 XP | 수용 인원 |
|---|---|---|
| L1 → L5 | ~500 | 15 → 20 |
| L6 → L10 | ~1,500 | 20 → 25 |
| L11 → L16 | ~4,000 | 25 → **30(상한)** |
| L17+ | (체증) | 30 고정 — 보상은 **버프·월드맵 혜택**으로 전환 |

- 누적 약 **34,000 XP → 활성 길드 기준 약 1개월에 수용 상한 30 도달**(길드가 커질수록 가속).
- 레벨 16 이후는 인원 대신 **길드 버프 / 월드맵 점령력·슬롯** 등으로 계속 성장(엔드게임).

## 3. 개인 기여도

- **누적 기여도 = 본인이 생성한 길드 XP와 1:1 누적**(기부 + 미션). 한 숫자로 투명 관리.
- 용도: ① **무임승차 판단**(길드 운영), ② **길드장 자동 위임 tie-break**(§4). (세금 분배는 기여도가 아니라 길드장의 균등/특정 선택 §5.5.)

## 4. 직책 / 운영

- **직책**: 길드장 · 부길드장(복수 가능) · 멤버. (권한 세부 TBD)
- **길드장 7일 미접속 → 자동 위임**:
  - 부길드장에게 위임. **복수면 누적 기여도 1위 → 동률 시 가입 오래된 순**.
  - 부길드장 전원 잠수/부재 → **동일 규칙으로 다음 활성 멤버 연쇄 승격**(빈 길드 영구 잠금 방지).
  - **5~6일차 경고 알림**(푸시 + 우편) 1회 후 7일차 위임 — 억울한 강등 방지.
  - 강등된 길드장은 **추방 아님, 멤버로 유지**(직책만 이전).
- 결정론(서버 타임스탬프) — 수동 개입 0.

## 5. 월드맵 — 지역 공격 / 수비

> 월드맵에서 구역을 공격·수비해 점령하는 길드 경쟁. 지역·점령단위·점령전·세금·데이터구조는 아래 확정. 시뮬 튜닝값과 일부 엣지는 각 절에 TBD로 표기.

### 5.1 지역 (레이드 5지역 모티브) — 확정

월드맵은 **레이드 지역을 모티브**로 한다(아트·로어·테마 재사용 → 신규 정체성 에셋 부담 0, 레이드↔월드맵 세계관 일관). 5지역 + 방위 배치:

1. **드래곤 화산** — 서(西)
2. **잊힌 신전** — 북(北)
3. **슬라임 늪** — 남(南)
4. **타락한 천사들의 하늘섬** — 중앙 (대륙 심연 위 부유섬, 부서진 신성 유적·천사상)
5. **오크 부락** — 동(東)

- **레이드 보스 1:1 매핑**: 드래곤화산=dragon_west · 잊힌신전=**stone_golem(신전 수호 석상 골렘)** · 슬라임늪=slime_king · 타락천사하늘섬=fallen_angel · 오크부락=orc_chief. (신전 보스 스프라이트를 더 신전 테마로 재생성할지는 선택 — 미정 시 현 stone_golem 유지.)

### 5.2 점령 단위 — 확정: 다중 거점, 총 50구역

- **(A) 다중 거점 방식**: 월드맵을 **총 50구역(노드)**으로 분할(외곽 4지역 각 11 + 중앙 6, §5.6 — 조정 가능). 여러 길드가 나눠 점령 → 희소성 해소.
- 구역 단위로 **공격/수비** 진행. 구역별 소유 길드·점령 상태를 보유.

### 5.3 지도 표현 (아이소메트릭 베이스맵 + 코드 노드 오버레이)

- **베이스맵**: 단일 아이소메트릭 픽셀아트 월드맵 1장 — `public/sprites/guild/worldmap.png`(1024², `image-rendering: pixelated`로 렌더). 5지역이 하나의 링형 대륙으로 자연스럽게 이어지고, **중앙에 타락한 천사들의 부유섬이 대륙보다 높게 떠 있음**. 어두운 톤·지역간 고저차·완만한 경사 경계.
- **제작 파이프라인**: 정밀 베이스를 외부 이미지 모델로 생성 → sharp로 워터마크 제거 + 팔레트 양자화(또렷한 픽셀 + 용량 최적). 지역마다 **작은 오브젝트(유적·캠프·용암구·슬라임풀)를 여러 규모로 분산** → 노드 거점 후보.
- **노드(50구역)**: 이미지에 굽지 않고 **코드 오버레이**(클릭 가능 + 소유 길드 색). 베이스맵은 배경만 담당.

### 5.4 점령전 (Conquest) — 확정 골격

- **점령 단위**: 구역(§5.2, 총 50구역). 길드가 구역 단위로 점령·소유.
- **공격/방어**: 중립·적 구역은 **공격**, 자기 점령 구역은 **방어** 가능.
- **전투 방식**: **별도 전투 엔진(팀전 결정론)**. 대난투의 결정론·리플레이·서버산출 모델은 차용하되, **길드=팀(아군 비공격)·최후 1길드 생존 판정**을 위해 대난투(개인전)와 **다른 별도 엔진이 필요**(재사용 아님). 배치 인원·전투력 기반, 실시간 PvP 아님.
- **전투 시각**: 매일 **KST 12:00~13:00** 1회. 12:00 배치 잠금 → 결정론 산출 → 결과·리플레이 공개·소유권 갱신.
- **배치(그 외 시간)**: 전투 시각 전까지 멤버를 **공격 인원**(목표 구역 지정) 또는 **방어 인원**(자기 점령 구역에만) 배치.
- **안개(은닉)**: 공격·방어 인원은 **자기 길드 것만 열람**. 적의 배치 규모·구성은 전투 결과 전까지 비공개(맵의 구역 소유 현황 자체는 공개).

**전제**(§0 연장, 유지):
- 결정론·비동기 / 반-스노볼(영주제 인력 상한) / 시간대 공정(KST 고정) / 공·수 비대칭(수비 프리미엄) / 무과금 공정(점령력 유료구매 금지) / 막판 갈아타기 차단(보상 자격 게이팅 §1).

**판정 규칙 — 확정**:
- **다중 길드 FFA**: 한 구역에 방어 길드 + 공격 길드 여럿이 **한 판에서 동시 전투**. **같은 길드원끼리는 공격 안 함**(길드=팀).
- **인접 규칙 없음**: 어느 구역이든 공격 가능(영토 연속성 제약 없음).
- **승리 조건**: 대난투식 결정론 배틀로열 — 전멸전으로 **마지막에 살아남은 인원이 한 길드뿐이면 그 길드 승리**(점령/유지). 무방비(방어 0) 구역은 공격 길드들끼리 가려 승자가 점령, 단독 공격이면 무혈 점령.
- **수비 프리미엄**: 일반 방어 인원 전투력 **+20%**, **영주 ×3**(방어 거점 앵커). 추천 시작값, 시뮬 튜닝.
- **멤버 배치**: 1인 1배치/일(공격 또는 방어, 한 구역에만). 구역당 인원 상한 없음(길드 30명 cap으로 자연 제한).
- **영주(대표)** — 확정: 점령 구역마다 길드원 **1명 이상 영주 지정 필수** → 보유 가능 구역이 인원에 따라 자연 제한(반-스노볼, 세수 상한 대체). 영주는 **배치 슬롯을 사용**(타 구역 공격 불가, **해당 구역 방어 전담**), **전투력 ×3**, **그 구역 세금 수집권 독점**(영주만 수집).
  - 영주가 슬롯을 점유하므로 **실보유 구역 ≈ 영주 가능 인원**(공격·방어와 인력 경합) → **한 길드의 다구역 점령은 의도적으로 어려움**(설계 목표).
- **영토 지속성**: **영구 지속**(시즌 리셋 없음). 장기적으로 선점 고착이 신규 길드 진입을 막으면 **시즌제 도입을 미래 옵션으로 검토**.

### 5.5 점령 보상 — 세금(Tax) 모델

영토 보유 이득을 **"세금"**으로 설계 — 게임 핵심 루프(강화)를 영토에 연결한다.

- **거주 구역**: 모든 유저가 머무를 구역 1곳을 **직접 선택**(최초 **랜덤 배정**, 이후 변경 자유·쿨다운 없음 §ⓑ).
- **포인트 발생**: 거주 구역의 유저가 **강화 성공 시에만**, **도달한 강화 레벨 = 가산 포인트**(예: +99 성공 → 99pt). 고레벨일수록 성공이 드물고 오래 걸려 **자연 가중**(고강화 1회 = 큰 포인트, 반복 파밍은 비현실적). 유저는 잃는 것 없음 — 활동의 부산물 산출.
- **수집(harvest)**: 누적 포인트는 **수집 쿨다운 1시간**으로 거두어 **소유 길드 풀**에 누적. **수집은 그 구역 영주만 가능**(§5.4).
- **환산·분배**: **100포인트 ≈ 1💎**(시뮬 튜닝 §ⓔ). 길드가 **균등 배분 / 특정 유저 지급** 중 선택 — **분배 권한 = 길드장**, **분배 내역 로그 공개**(리더 독식 견제).
- **효과(메타)**: 활동 많은(거주자 많은) 지역일수록 세수↑ → 길드는 **인구 많은 지역을 노림**, 유저는 **자기 길드 영토에 거주**해 길드를 살찌움 → 가입·영토·활동이 한 고리로 연결.

**세금 모델 — 확정**:
- ⓐ **거주자 = 전 유저**(비길드 포함). 누구의 활동이든 그 땅 소유 길드가 과세 → 영토에 실가치.
- ⓑ **거주 변경 즉시 가능(쿨다운 없음)**. 포인트는 유저가 아닌 **지역**에 누적되고 점령 시 이전되므로 "들고 튀기" 악용 불가 → 이동 자유.
- ⓒ **점령 시 미수집 포인트 유지 → 신 소유자 이전**(약탈형). 전투 시각(12:00)이 고정이라 **그 전에 수집하면 손실 없음** → 방어측은 전투 전 수집이 정석.
- ⓓ **세수 상한 없음.** 대신 **점령 구역마다 영주(대표) 1명 이상 지정 필수**(§5.4) → 멤버 수(30 cap)가 **자연 상한**(여러 구역 동시 점령은 인력상 어려움).
- ⓔ **환산율 시뮬 튜닝**: 기본 100pt:1💎, 세수가 과하면 **1000:1 등으로 하향**해 경제(1,440💎/일 앵커) 교란 방지. 💎는 ratio로 캡되는 소량 faucet.

### 5.6 데이터 구조 (DB 설계)

> Drizzle 스키마 + 수동 마이그레이션으로 구현 예정. 구역은 좌표 + (미사용)인접 그래프를 가진 평면 리스트.
> **구역 수: 총 50 확정** — 외곽 4지역 각 11 + 중앙(천사섬) 6 = 50(제안 배분, 조정 가능). 지역별 개수 달라도 무방, `zones` 행 추가로 확장 자유.

**A. 길드 코어**
- `guilds`: id, name(변경 불가), emblem_url(키워드 조합 생성물 §1.6), emblem_color(맵 틴트용 대표색), notice(≤60자), level, xp, leader_user_id, created_at
- `guild_members`: guild_id, user_id, role(`leader|vice|member`), contribution_points(누적 기여도), joined_at, daily_donation_count, last_donation_kst_day
  - 유니크(user_id) — 1유저 1길드. 탈퇴 24h 재가입 제한은 별도 `guild_leave_log(user_id, left_at)` 또는 멤버 행 soft-delete + 타임스탬프.

**B. 영토 / 구역**
- `zones`(구역): id, region(`volcano|temple|swamp|sky|orc`), name, map_x, map_y(오버레이 좌표), owner_guild_id(nullable=중립), lord_user_id(nullable, 영주 상시 지정), captured_at, tax_points(미수집 누적), last_tax_collected_at(1h 쿨다운)
  - 영주(lord_user_id)는 **상시 필드**(전투 외에도 세금 수집·자동 방어). 소유 시 ≥1 필수.
- `zone_adjacency`(인접 그래프 — **현재 미사용·미래 대비**): zone_a, zone_b(무방향 간선, 정규형 `a<b`). 좌표 배치 시 함께 author.
  - **현 FFA 점령전은 인접 무시**(어디든 공격). 정의만 보유 → 후일 ① 확장형 점령(인접만 공격) ② 맵 연결선 표시 ③ 경로/그룹핑 전환 시 즉시 활용. 코드/규칙에 결합하지 않아 유지비 0.

**C. 거주 / 세금**
- 거주(**구역 단위 확정**): `profiles.residence_zone_id`(유저당 1구역, 최초 랜덤, 이동 자유). 강화 성공 시 `zones[거주].tax_points += 성공단계`(강화 resolve 훅에서 가산).
- 길드 세금 풀: `guilds.tax_pool_points`(영주가 수집 시 zone→길드 이전). 분배 액션이 `100:N💎` 환산 후 균등/특정 지급(기존 다이아 grant·우편 재사용).

**D. 일일 배치 / 전투**
- `guild_battle_deployments`: id, battle_kst_day, user_id, guild_id, zone_id, role(`attack|defend`), created_at. **유니크(user_id, battle_kst_day)** = 1인 1배치. 12:00 잠금. (영주는 자기 구역 방어로 자동 포함 → 별도 공격 배치 불가.)
- `conquest_battles`: id, battle_kst_day, zone_id, winner_guild_id, finale(jsonb: 참가자·전투력·리플레이), created_at. **유니크(zone_id, battle_kst_day)**. **별도 팀전 엔진**(§5.4 — 영주 ×3, 일반 방어 +20%, 같은 길드 비공격, 최후 1길드 생존=승).

**재사용·통합 지점**: 전투력 산식 = 공유 / 결정론·리플레이 **모델 차용**하되 점령 전투는 **별도 팀전 엔진**(대난투=개인전과 다름, §5.4) / 💎 지급 = 기존 grant·mail / 거주 세금 가산 = `enhance/resolve` 훅(성공 시 도달 레벨 가산) / 정산 cron = KST 12:00 / 신규 유저 가입 시 `residence_zone_id` 랜덤 배정.

**구현 TBD(엣지)**: 점령 직후 영주 미지정 시 처리(유예/중립 복귀), 영주 잠수·탈퇴 시 승계, 거주 구역이 소멸/리밸런싱될 때 마이그레이션. (길드 해산 시 구역 중립화·세금 풀 소멸은 §1에서 확정.)
