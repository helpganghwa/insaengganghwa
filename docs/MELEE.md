# MELEE — 대난투 (Grand Melee)

> 전 유저가 매일 자동 참가하는 결정론 배틀로열. 강화/초월로 쌓은 전투력이 순위로 환원되는 일일 경쟁·복귀 의례. 서버가 전 배틀을 결정론으로 산출하고 클라는 재생만 하므로 실시간 PvP 인프라·치트 위험이 없다(1인 운영 친화).

관련: `GDD.md`(코어 루프) · `BALANCE.md §3`(전투력) · `SCHEMA.md`(스키마) · `MAIL.md`(결과 우편) · `PROFILE.md`(아바타 — 리플레이 등장)

---

## 1. 개요

- **자동 참가**: 별도 신청 없음. 조건 충족 유저는 매일 자동 출전.
- **일일 1회 정기 이벤트**(KST): 자정 명단 확정(무알림) → 오전 9시 결정론 산출(비공개) → 9~10시 진행(난투 → 우승컵 전달) → **10:00 결과 발표·리플레이 공개**.
- **결정론 산출 + 온디맨드 재생**: 9시 cron이 전 배틀을 한 번에 계산(결과 비공개, 9:00~10:00 진행 대기 — 9:00~9:45 "난투 진행 중" → 9:45~10:00(잔여 15분) "우승자에게 우승컵 전달 중"). **10:00 발표 후** 온디맨드 재생. 재생은 **전투 리플레이(전체 or 마지막 N라운드) + 내 전투(개인 리캡)** 2탭(§8) — 총 라운드 ≤`MELEE_REPLAY_ROUNDS`(=1000)면 전체, 초과면 마지막 그만큼만 저장(링 버퍼)해 어떤 N에도 일정.
- **무료 진입 고정** — 참가·순위에 재화 투입 경로 없음. 결정론·무과금이라 게임산업법 §33(확률형) 비대상. ⚠ "돈 내고 대난투 유리" 류는 절대 도입 금지(도입 즉시 §33 리스크).

---

## 2. 참가 / 로스터

- **대상**: 가입 후 **강화 1회 이상** 경험한 유저 전원(`enhancement_logs` 1건+). 순수 설치·미플레이 계정은 제외(noise 방지).
- **명단 락**: 매일 **KST 자정** 시점 대상자로 로스터 확정. 자정 이후엔 그날 대난투에 신규 합류 불가.
- **단일 글로벌 풀 고정**: 룸 샤딩·디비전(전투력 구간 분할) 모두 없음 — **1천만 명이어도 하나의 난투·하나의 랭킹·하나의 챔피언**(설계 결정). 스케일은 finale-only 리플레이 + 배치로 흡수(§9).

## 3. 일정 (KST)

| 시각 | 처리 |
|------|------|
| **00:00** | 로스터 락 — 그날 참가자 명단 확정. **무알림(조용)** |
| **09:00** | 참가자 **전투력 스냅샷** + 전 배틀 결정론 산출(cron) → 로그/결과 저장. **결과 비공개**, 상태 "난투 진행 중" |
| **09:00~09:45** | **"난투 진행 중" 대기 UI**(결과·리플레이 비공개). 타이머 1:00:00부터 카운트다운 |
| **09:45~10:00** | (잔여 15:00) **"우승자에게 우승컵 전달 중"** — 라벨만 전환, 같은 타이머. 우승컵 연출 버퍼(우승자 포즈 생성 구간) |
| **10:00** | **결과 발표** — 결과 우편/푸시 발송 + 순위·리플레이 공개(우승자는 우승컵 포즈로 등장). **재생은 10:00부터** 가능 |
| 종료 후 | 하루 종일 **온디맨드 리플레이 + 결과 조회**(관람은 보너스, 보상은 비동기 수령) |

- **명단 = 자정 락 / 전투력 = 9시 락**(자정~9시 사이 강화 진행분도 반영).
- 결과는 9시에 이미 산출되어 있으나 **10:00까지 비공개**(발표 긴장감 + 우승컵 연출 버퍼). 9:00~10:00은 라이브 중계가 아니라 "난투 진행 중 → 우승컵 전달 중" 대기 화면.
- 10:00 발표는 무관전이어도 손해 없음 — 순위·보상은 우편/푸시 비동기, 리플레이는 종일 조회.

## 4. 전투 모델 (체인 난투)

배틀 시작 시 각 참가자: **HP = 전투력 × `MELEE_HP_MULT`(=2)**. HP는 배틀 내내 유지되며 피격 시 누적 차감(소모전).

라운드 반복(생존 1명 남을 때까지):

1. **공격자 선정** — 직전 라운드 생존자(반격 체인). 체인 시작(킬 직후 또는 최초)이면 **생존자 중 완전 랜덤**.
2. **타겟 선정** — 생존자 중 **공격자 제외 완전 랜덤**(균등).
3. **타격(항상 명중)** — `데미지 = round(공격자 전투력 × U(0.5, 1.2))`. 타겟 HP에서 차감.
4. **판정**
   - HP ≤ 0 → **탈락**. 등수 = 현재 남은 최저 등수(첫 탈락 = N위 → N−1위 → …). 체인 종료 → 다음 라운드는 새 랜덤 공격자.
   - HP > 0 → **버팀**. **그 타겟이 다음 공격자**가 됨(생존자 반격 체인).
5. 마지막 생존자 = **1위**.

**설계 의도 / 밸런스**
- 전투력 → HP 매핑이 등수의 주동력(잘 버팀). 체인 구조상 잘 버틴 자가 공격권을 얻어 공격력도 보조로 보상 — 전투력 양축 보상.
- `HP_MULT=2` + 데미지 평균 0.85 → 동급 hits-to-kill ≈ 2.35. 시뮬(`scripts` 몬테카를로) 근거: bottom-heavy 단일 풀에서 상위 전투력 보호(첫탈락·추락 ≈ 0~0.3%), 동시에 중상위권 업셋 여지 유지. 더 높이면 더 길고 더 뻔해질 뿐이라 ×2 채택.
- 평균 ~배수×N 라운드(예 50명 ≈ 100라운드) — 로그/리플레이 길이와 직결.
- **모든 RNG는 서버 결정론**(아래 §5). 클라는 결과 재생만.

> 상수(`MELEE_HP_MULT`, 데미지 분산 `0.5~1.2`)는 `lib/game/balance.ts`에 박제 — 본 문서와 1:1.

## 5. 결정론 / 안티치트

- 9시 cron이 **단일 시드(예: KST 날짜 기반)**로 공격자/타겟/데미지 롤을 순서대로 결정 → 전 배틀 산출. 동일 입력(로스터+CP 스냅샷+시드)이면 항상 동일 결과(재현 가능).
- 산출 결과는 **순서가 있는 이벤트 로그**로 저장. 클라는 10:00 발표 후 이 로그를 **온디맨드 재생만** 하고 **결과를 계산하지 않는다**.
- 전투력 스냅샷은 9시에 서버에서 산정(보유 카탈로그 중복제외 최고 인스턴스 × 초월배수, `BALANCE §3.2`). 클라 보고값 불신.
- **조기 누출 방지**: 9시에 산출돼 있어도 결과·로그·순위 API는 **10:00 전까지 비공개**(status='computed'→'revealed' flip = 발표 cron). 발표 전 조회 차단.

## 6. 보상

등수 결정론. **메일 지급 + 앱 푸시**(§7). 상자는 3슬롯 균등(무기/방어구/장신구 동일 — 개수가 3의 배수라 정확히 1/3씩).

| 등급 | 기준 | 다이아 | 보급 상자 |
|------|------|--------|-----------|
| 🥇 1위 | 1위 | 💎2,000 | 60 |
| 🥈 2~3위 | 2~3위 | 💎1,000 | 30 |
| 정예 | 상위 5% | 💎400 | 15 |
| 상위 | 상위 20% | 💎150 | 15 |
| 중위 | 상위 50% | 💎100 | 9 |
| 참가 | 나머지(전 참가자) | 💎50 | 6 |

- **티어 배타 적용**(등수 순): 1위 → 2~3위 → 다음 상위5% → 20% → 50% → 나머지. 한 명은 가장 높은 해당 티어 **하나만** 수령.
- **소규모 N 처리**: 절대등수(1·2~3위) 우선 적용 후 나머지를 % 내림으로 배분. 임계값은 N 기반 계산(겹침 없음).
- **경제 영향**: 대난투가 **메인 다이아 획득 채널**(passive 축소 보완). floor(전 참가 💎50)가 평균을 지배, 1위 💎2,000은 글로벌 1명/일 희소상 → 평균 ~95💎/인·일. 박스는 초월 자동 진행 연료(랜덤 150종 희석이 천장).

## 7. 알림 (결과 발표)

- **10:00 발표 시에만** 참가 전원에게 **결과 우편**(`MAIL.md`) + **앱 푸시** 발송. (자정 명단 락·9시 산출은 무알림)
- 내용: **내 순위 + 챔피언(1위) 닉네임** 노출. 예: "오늘 대난투 **7위**! 👑 챔피언: 강화의신". (별도 칭호 시스템 없음 — 닉네임 호명으로 명예 부여)
- 푸시 카테고리 `push_melee` — **상시 발송**(일일 복귀 푸시 역할 겸함, 설정 토글 없음·끄기 불가).
- 보상은 우편 첨부(다이아+상자)로 비동기 수령 — 9시 관전 여부와 무관.

## 8. UI

진입: 홈 메뉴 **대난투** 카드 → `/melee`. 배경: 콜로세움 픽셀아트(`public/sprites/hub/melee.png`).

- **상태별 표시**:
  - 9시 이전: **"오전 9시 개시"** + 9시 카운트다운 (상단 무대 이미지 + 하단 보상표·역대 우승자)
  - 9:00~9:45: **"난투 진행 중"** 대기 화면 — 타이머 1:00:00부터 카운트다운
  - 9:45~10:00: **"우승자에게 우승컵 전달 중"**(잔여 15:00) — 같은 타이머, 라벨만 전환
  - 10:00 이후: **최종 랭킹(1·2·3위 강조) + 내 순위/보상** + 아래 2개 탭. (발표 지연 시 "우승자에게 우승컵 전달이 늦어지고 있습니다" + 경과 시간 0부터 카운트업)

- **2개 탭** (10:00 발표 후):
  - **전투 리플레이** — 콜로세움 배경 + 각 유저 **아바타** 공격/피격 **애니메이션 리플레이**(`finale` 로그, `PROFILE.md` 아바타). **총 라운드 ≤1000이면 전체, 초과면 마지막 1000라운드**(필드가 수백 명→챔피언으로 좁혀지는 클라이맥스). 전투 로그 리스트("누가 누구를 N 데미지 · 버팀 / 체력 바닥나 패배") 클릭 → 해당 장면 재생, 자동 페이싱(초반 빠르게→클라이맥스 느리게). *애니메이션은 fast-follow — v1은 로그+랭킹 우선.*
  - **내 전투 보기** — 개인 **리캡**(풀 애니메이션 아님): 내 등수 · **내가 쓰러뜨린 상대 목록**(`killer_user_id` 역참조) · **나를 쓰러뜨린 자** · 보상. 비-상위권 유저의 개인 서사. 스케일 무관(저장 거의 0).

## 9. 구현 메모 (미확정 / 후속 결정)

- **스키마**(`SCHEMA §13` / `schema/melee.ts` / `manual/0017_melee.sql`): `melee_battles`(date UNIQUE·seed·status·champion·**finale jsonb**) + `melee_participants`(battle·user·cp_snapshot·final_rank·**killer_user_id**·reward). 룸 없음. 멱등(date UNIQUE).
- **리플레이 전략**: 전체 이벤트 저장 불가(10M=2천만 이벤트) → **마지막 `MELEE_REPLAY_ROUNDS`(=1000) 라운드만 링 버퍼로 저장**(총 라운드 적으면 전체). `finale`={roster(등장 유저 닉·전투력·등수)+events(로컬 인덱스 압축 [a,t,d,k])}, ~수십 KB 상수. "내 전투"는 **개인 리캡**(등수·killer·내가 처치한 목록 역참조)으로 무저장 처리 — 풀 개인 애니메이션은 스케일상 미지원. (개인 킬수·데미지 스탯 컬럼은 후속 옵션)
- **Cron**: 자정 로스터 락(KST `0 15 * * *`) + 9시 산출·저장(KST 09:00 = UTC `0 0 * * *`) + **10:00 발표(KST 10:00 = UTC `0 1 * * *`)** — 결과 공개 flag + 우편/푸시. 인증·청크는 기존 cron 패턴 재사용. (lazy 보강 가능 — 조회 시 시각 지나면 공개)
- **스케일**: **단일 글로벌 풀 고정**(룸 샤딩·디비전 없음). 시뮬은 O(N) 메모리/시간, finale·내전투 리캡은 N무관. 초대규모(수백만+) 시 무거운 부분은 "9시 전 참가자 CP 일괄 산정 + N개 결과행 + N개 우편/푸시" → 집계 쿼리 1방·청크·배치 큐로 분산(현실 규모는 단일 크론으로 충분, 극단 스케일은 배치 파이프라인).
- **리플레이 포맷**: 이벤트 로그(공격자·타겟·데미지·결과)만 저장 → 10:00 이후 클라 온디맨드 재생(라운드당 페이싱은 클라 연출). 동기 재생 아님.
- **전투력 산식**: `BALANCE §3.2` 총전투력 그대로 사용. 데미지/HP 상수는 `balance.ts` 박제.
- **보존**: 배틀 로그·결과 N일 보관 후 prune(우편은 별도 만료).
