## 0. TL;DR

**한 줄 평**: 도덕적·기술적으로 잘 설계된 1인 운영 모바일 웹 게임. **컨셉/코드 품질은 9/10**, 그러나 **상업적 BM 깊이는 4/10 — 현 설계로는 1인 생계비 자력 충당이 어렵다**.

| 영역 | 점수 | 핵심 메시지 |
|------|------|----|
| 컨셉/기획 | A− | "+99 자랑" 한 줄로 모든 시스템이 정렬됨 |
| 기술 구현 | A | 1인 5년 운영을 지지하는 안티치트·법규 준수 인프라 |
| 바이럴 설계 | B+ | 카톡 OG·구경 모드는 영리, 보상 강도가 약함 |
| BM 깊이 | C | 단일 재화·천장 없음·보호권 없음·광고 없음 = 매출 동기 1개뿐 |
| 시장성 | C+ | 타겟이 좁고 (KR 20~40대 카톡 유저) 경쟁 IP 대비 인지도 0 |

**예상 12개월 누적 매출(현 설계 그대로 출시 가정)**:
- 비관(DAU 100): **₩1.5M–3M**
- 중립(DAU 500): **₩15M–35M**
- 낙관(DAU 2000): **₩80M–180M**

**1인 운영자 생계 손익분기점(월 ₩4M 순수익)**: **DAU 3,000–5,000 + ARPPU ₩40,000+ + 결제율 3%+** 동시 달성. 현 BM 설계로는 도달 불투명.

## 3. BM 설계자 관점

### 3.1 잘한 점

| # | 항목 | 평가 |
|---|------|------|
| ① | **단일 재화(다이아) 단순성** | UX 마찰 최소화. 환산률 1다이아=1분 단축으로 가치 인식 명확 |
| ② | **환불 보호·미성년 한도 사전 설계** | 법적 리스크 헷지. 후행 비용 0 |
| ③ | **포트원 + 카카오페이/토스/카드 = 한국 결제 표준** | 결제 전환율 ↑ |
| ④ | **확률 공시 = 코드 1:1** | §33 위반 시 형사처벌 위험 0 (이번 패치도 1:1 유지) |
| ⑤ | **분해 = 고정 10다이아** | 확률형 폐기로 합법성 ↑, UX 예측성 ↑ |
| ⑥ | **시즌제 없음**(누적 랭킹) | 운영 부담 ↓ |

### 3.2 못한 점 — BM 깊이의 구조적 결함

| # | 항목 | 문제 | 매출 영향 |
|---|------|------|----------|
| ① | **매출 동기가 1개뿐(시간 단축)** | 한국 RPG의 4대 매출 기둥(① 강화 보호권 ② 가챠 ③ 외형/코스튬 ④ 시즌 패스) 중 **0개 채택** | ARPPU 한계 ₩20,000/월 추정 |
| ② | **+99 = 4주 풀단축 ₩28,000** | 한 슬롯 +99을 즉시 완성하려면 4주 × 1440분 ≈ 40,000다이아 = ₩40,000. 6슬롯 × 사이클 무한 진행을 고려해도 paywall 천장이 너무 낮음 | 헤비 유저 매출 캡 |

| ④ | **가챠 결합 매출 없음** | 보급 상자가 IAP 패키지에 안 묶임. "다이아 10000 + 무기 상자 10개" 같은 결합 상품이 없어 ARPU 다양성 ↓ |
| ⑤ | **배틀 패스/구독 없음** | 월 ₩9,900 부담 적은 정액제 = idle 게임의 안정적 매출원. 미설계 |
| ⑥ | **광고 매출 0 — 보류 확정** | 모바일 웹은 보상형 광고 SSV 부재로 어뷰징 ROI 통제 불가. 보상형/배너 모두 v1 미도입 결정(2026-05-25). Capacitor wrapper 시점에 재검토하되, 그 이전에는 매출 모델에서 광고 컬럼 제외 |
| ⑦ | **레이드 호스트 비용 1000다이아의 단방향성** | 호스트가 비용 부담, 9명 무료 참여 = 무임승차 구조. 호스트 LTV 손상 |
| ⑧ | **닉네임 변경 1000다이아** | 단발성·소액. BM 효과 미미 |
| ⑨ | **F2P 결제 전환 트리거 없음** | Day1 무료 5다이아 외, 결제 동기를 만드는 "결정적 순간 paywall"(예: +52 첫 하락 시 단축 제안) 없음 |

### 3.3 BM 개선 — 가장 중요한 8가지

#### P0 (출시 전 필수, 매출 직접 영향)

**B0-1. 배틀 패스 도입 — 월 ₩9,900, ARPPU 안정화**
- 시즌(30일) 단위, 무료 트랙 + 유료 트랙
- 일일/주간 미션 클리어로 경험치 → 패스 단계 보상
- 유료 트랙: 다이아 +5000, 보급 상자 30개, 한정 칭호/프레임
- 패스 구입 유저 = D30 유지율 4배 (업계 표준)

**B0-2. 패키지 보너스 곡선 가속**
```
현행:  starter 0% → mega 28%
권장: starter 0% → mega 100%+ (한국 표준)
```
- medium(₩13,000) 권장: 14,000 → 18,000다이아 (+40%)
- mega(₩65,000) 권장: 78,000 → 130,000다이아 (+100%)

**B0-3. 결합 상품 (다이아 + 보급)**
- "강화의 시작" (₩4,900): 다이아 5,000 + 보급 상자 5개 (슬롯 선택)
- "주말 패키지" (₩19,900): 다이아 25,000 + 보급 30개 + 분해 부스터(48h 2배)
- 결합 상품은 일반 패키지 대비 전환율 1.8배 (한국 모바일 RPG 평균)

#### P1 (출시 +1개월 내)

**B1-1. "안전 강화권" 도입 — 보호권 없음 원칙의 유연한 우회**
- 보호권/축복권은 안 도입하지만 "강화 보석 ×3 단축권" (₩2,900/회 또는 다이아 3000)
- +52 첫 하락 직전 결정 모달에 paywall 자연 노출
- 게임산업법 우회: 확률 변화 아님, 시간 단축 가속 = 합법

**B1-2. 시즌 보스 + 한정 코스튬**
- 3개월마다 신규 보스 + 한정 무기 외형(성능 동일, 외형만 차별)
- 시즌 보스 도전 패키지 ₩14,900: 추가 공격 50회 + 한정 무기 보장
- "등급 없음" 원칙 유지하면서 외형 차별화 매출 신설

## 6. 결론 및 권장 사항

### 핵심 메시지

**인생강화는 기획·기술적으로 "도덕적이고 잘 만든 게임"이지만, 현재 BM 설계로는 1인 생계비를 충당할 매출에 도달하기 어렵다.** 출시 전 BM 깊이를 한 번 더 다듬어야 한다.

### 출시 전 필수 (P0) 5개

1. **배틀 패스 ₩9,900/월** — 매출 안정화 핵심
2. **패키지 보너스 비율 2배 상향** — 한국 시장 표준 정렬
3. **결합 상품 3종** (다이아+보급) — ARPU 다양화
4. **D3/D7/D14 로그인 캘린더** — 유지율 방어
5. **추천 보상 ₩300 → ₩1,500** — K-factor 개선

### 출시 +3개월 (P1) 2개

1. **PWA 설치 prompt + TWA Play Store 진입** — 신규 채널
2. **"강화 가속권" ₩2,900 (시간 단축, 확률 무관)** — 결정적 paywall

> ~~광고 SDK 도입~~ — 보류 확정(2026-05-25). 모바일 웹 SSV 부재 + 어뷰징 ROI 통제 불가. Capacitor wrapper 완료 후 재검토(P2 후속).

### 장기 (P2) 2개

1. **길드 시스템 + 길드 패스** — 사회적 동기 + 정액 매출
2. **분기 시즌 보스 + 한정 외형** — 컨텐츠 호흡

### 한 줄 결론

> 컨셉과 코드는 "장인 작품" 수준이지만, **BM은 "취미 프로젝트" 수준에 머물러 있다.** 위 P0 5개만 적용해도 12개월차 매출 기댓값이 **2~3배** 상승하고, **DAU 2,400**에서 1인 생계가 가능해진다. 적용 안 하면 DAU 6,800이 필요한데 — 이는 한국 모바일 웹 게임 단독으로 도달하기 매우 어려운 숫자다. 광고 매출은 모바일 웹 SSV 부재로 보류 확정 — Capacitor wrapper 시점까지는 IAP·배틀패스·결합상품 3축으로만 수익을 만든다.
