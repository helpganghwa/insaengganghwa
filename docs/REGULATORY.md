# REGULATORY.md — 법규·등급분류·합법 출시 단일 출처

> 인생강화(웹/PWA 모바일 게임, 한국 서비스)의 출시·운영에 필요한 법적 요건을 한 곳에 모은다.
> 재미·기능과 무관하게 **합법 출시의 전제조건**이며, 결제·확률·연령이 얽히므로 코드·약관·운영이
> 이 문서와 어긋나면 안 된다. 법령 해석은 시점 의존적 — 신청 직전 GCRB 상담·게임 전문 변호사
> 검토로 최종 확인한다. (기준 시점: 게임산업법 확률표시 의무 시행 후, 등급분류 GCRB 일원화 후)

---

## 0. 핵심 원칙

1. **유통 전 등급분류 필수.** 등급분류증명서 없이 공개 서비스 불가(게임산업법). 정식 등급 전까지
   프로덕션은 **비공개/제한 베타**로 둔다(공개 URL = 유통으로 간주될 위험).
2. **확률공시 = 코드와 1:1.** `BALANCE.md` ↔ `lib/game/balance.ts` ↔ `probability_snapshots`.
   불일치는 형사처벌 위험(게임산업법 §33). CLAUDE.md §3.5와 동일 규율.
3. **연령등급에 맞춘 청소년 보호.** 청소년이용가 목표 시 본인인증·미성년 결제 보호가 등급 조건.
4. **결제는 전자상거래법 준수.** 청약철회·환불·영수증 고지 의무.

---

## 1. 등급분류

### 1.1 주체·경로
- **GCRB(게임콘텐츠등급분류위원회)** 가 전 연령등급(청소년이용가~청소년이용불가) 등급분류를 담당.
  게임물관리위원회(게임위)는 사후관리·감독.
- 본 게임은 **웹/PWA**라 마켓 자체등급분류(구글플레이·애플·원스토어 설문) 경로 밖 → **GCRB 직접 신청**이 기본.
  추후 PWA를 오픈마켓에 앱으로 등재하면 해당 마켓의 자체등급분류 경로를 병행할 수 있다.

### 1.2 예상 등급
- 강화·보급에 **확률형 요소 + 유료 결제**가 있으나 **획득물이 게임 내 재화이며 환전 불가** → 사행성 낮음.
- 목표 **청소년이용가(12/15세)**. 전체이용가는 결제·확률형 때문에 어려울 수 있음. 확정 등급은 심의 결과.

### 1.3 절차
1. 사업자 등록(개인사업자 가능) — 신청 주체.
2. 심의용 접근 환경: 전체 콘텐츠를 볼 수 있는 URL + **재화가 충전된 테스트 계정**.
3. 게임 내용 설명서(시스템·결제·확률형 요소·연령 보호) 제출.
4. GCRB 등급분류 신청 → 심의(통상 10~15일) → **등급분류증명서** 발급.
5. 게임 내·서비스 노출 화면에 등급 표시 → 정식 오픈.
- 수수료: 등급·분량별 차등(청소년이용가 상대적 저렴).

## 2. 확률형 아이템 확률 공시 (게임산업법 §33)

### 2.1 해당성
- **강화**(유료 다이아로 시도, 성공/유지/하락이 확률로 결정)와 **보급/상자**는 확률형 아이템에 해당.
  → 확률 구성비·산정기준 표시 의무 대상.

### 2.2 면제 기준
- **3년 연평균 매출 1억원 이하 중소기업**이 제작·배급하는 게임물은 표시 의무에서 제외.
  신규·매출 0 단계인 본 게임은 면제 대상에 들 가능성이 높다.
- **단, 면제돼도 자율 공시를 유지한다.** 이미 시스템이 구현돼 있고(아래), 매출이 면제선을 넘는
  순간 의무가 발생하므로 처음부터 켜두는 것이 신뢰·리스크 양면에서 유리.

### 2.3 현 구현 (유지·검증 대상)
- `lib/game/balance.ts` — 모든 확률/시간/비용의 단일 진실 원천(`BALANCE.md`와 1:1).
- `probability_snapshots` — 확률 변경 시 영구 기록. 변경 시 **사이트 공지 24시간 사전 고지**.
- 공시 페이지(동적 생성)가 현행 확률을 그대로 노출 — 코드값과 표시값이 항상 일치해야 한다.

## 3. 본인인증·연령 보호

- **PASS/KMC 본인인증** 연동 — 결제 모듈과 함께 진행(카카오 OAuth 단독으로는 연령·실명 미확보).
- 청소년이용가 등급 시 요건:
  - 미성년자 식별(생년) 및 **법정대리인 동의**·미성년 결제 취소권 안내.
  - 미성년 **월 결제 한도** 적용(법정 한도 준수).
- 저장 최소화: 생년 등은 해시/검증값 위주(`profiles.birth_year_hash`·`identity_verified_at`).
- **만 14세 미만 가입 게이트(결정 2026-07-10)**: 약관 제4조("만 14세 미만 가입 불가")의 실확인은
  **결제 시점 본인인증으로 갈음**한다 — 가입(카카오 로그인) 시점에는 연령 확인 없음(카카오
  OAuth가 연령을 제공하지 않고, 가입 자체는 무료 이용이라 개인정보 최소수집 원칙 우선).
  결제·유료 이용 진입 시 PASS/KMC 본인인증이 연령을 실확인하고 미성년 보호(월 한도)를 적용.
  등급분류 결과가 연령 확인 강화를 요구하면 재검토.

## 4. 결제·환불 (전자상거래법)

- **포트원** 결제 연동(카카오페이/토스/카드). 영수증은 **PG 결제모듈의 이메일 입력·매출전표 발송으로 갈음**(자체 발송 없음) — 발급 의무 충족 경로를 PG에 위임.
- **청약철회/환불**: 미사용 유료 재화는 결제 후 7일 이내 청약철회 가능(고지 의무). 사용분·소진분 처리
  기준을 약관에 명시.
- 결제 멱등: `portoneOrderId` UNIQUE(이중 결제 방지) — CLAUDE.md §3.4.
- 결제 화면·구매 전 **확률 정보 및 환불 정책 접근 경로** 노출.

| 문서 | 의무 | 현황 |

| 이용약관 | 필수 | **초안 게시** `/legal/terms` (lib/legal/content.ts) |

## 6. 출시 합법화 체크리스트 (순서)

```
① 사업자 등록
② 약관 2종(이용약관·개인정보처리방침) + 환불정책 + 사업자정보 표시
③ 포트원 결제 + PASS/KMC 본인인증 연동(미성년 보호·결제한도 포함)
④ 심의용 테스트 계정·게임 내용 설명서·확률표 정리
⑤ GCRB 등급분류 신청 → 등급분류증명서
⑥ 등급 표시 후 정식 오픈(그 전까지 프로덕션은 제한 베타)
```

- ②~④는 등급분류 신청 시 함께 보이는 게 깔끔 → **결제 트랙과 묶어 한 번에** 진행.
- 정식 오픈 전 공개 URL은 접근 제한(초대/베타 플래그)으로 "유통" 리스크를 차단.

## 7. 변경 관리

- 확률·비용 변경: `BALANCE.md` 갱신 → `balance.ts` 동기 → `probability_snapshots` 기록 → 24h 사전 공지.
- 등급에 영향 줄 콘텐츠 변경(과금 구조·확률형 신설·연령 요소)은 **재등급 필요성**을 먼저 검토.
- 법령은 개정될 수 있다 — 매출이 확률공시 면제선(연평균 1억)을 넘기 전, 면제 종료에 대비한다.
