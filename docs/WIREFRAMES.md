## 2. 강화 (코어 화면 — grow 실제 구조 그대로)

**grow의 실제 화면(`EnhanceSlotCard`)을 그대로 이식**한다. 레이아웃·인터랙션·연출 동일, **등급 → 초월 치환** + **파괴·위험경고 삭제**만 적용. 3슬롯 × 부위당 2 lane(총 6 카드), 각 카드 높이 고정(100px, CLS 0).

### 2.1 활성 lane 카드 (grow 구조)

```
┌─[ 테두리 = 진행 게이지(둘레 차오름, 색: 적<33%<황<66%<녹) ]─┐
│ [스프라이트] 룬소드  ✦T3   +47→+48                  💎 72 │
│  42px·초월   성공 38.0%   최대 50%                    취소 │
│  테두리/오라  하락 4.2%                                     │
│              최대 확률까지 2시간 14분                       │
└────────────────────────────────────────────────────────────┘
   카드 본문 탭 → "강화하시겠습니까?" 컨펌(항상) → 강화 시도
```

- **테두리 게이지**(BorderGauge): 카드 외곽선이 진행도(경과/총)를 둘레 따라 채움. 색 = 진행도(적→황→녹). 완료 시 글로우 강조
- **본문 탭 = 강화 시도**: 탭 시 항상 인라인 컨펌 — 완료 전이면 "아직 최대 확률이 아닙니다. 그래도 강화하시겠습니까?", 완료면 "강화하시겠습니까?" → [강화]/[취소]
- **좌측**: 스프라이트(42px). grow의 *등급 테두리(GRADE_BORDER_CLASS)* → **초월 단계 테두리/오라(✦T0~T10)**로 치환
- **중앙 4행**:
  1. 아이템명 · `✦T{초월}` 배지(= grow의 *등급 라벨 GRADE_LABEL* 자리) · `+from→+target`
  2. `성공 X.X%`(effective=base×경과÷총, BALANCE §1.2) · `최대 Y%`(등록 시 baseRate 스냅샷)
  3. `하락 X.X%`(= grow의 *하락/파괴 2행* 중 **하락만**, 파괴 행 삭제. +0~+51은 "안전 — 유지" 표기)
  4. `최대 확률까지 {남은시간}` / `강화 가능` / `강화 중…`
- **우측**: `💎{즉시 단축 비용}`(1다이아=1분, BALANCE §6.2; 완료 시 "완료" 비활성) · `취소`(2단계, 3초 카운트다운 후 확정 → lane 즉시 해제·무료)
- **결과 플래시**: 시도 후 카드에 로어 문구 1.6초 오버레이 + 효과음/햅틱. 톤: 성공(녹)/유지(회)/하락(황). **파괴(적) 삭제**
- **자동 재등록**: 성공 시 다음 레벨 자동 큐잉(grow `tryAutoEnqueue`). 단, grow의 *위험 임계 동의(InlineRiskyAck/RISK_ACK)* 흐름은 **전면 삭제**(경고 없음, GDD §3.2)

### 2.2 빈 lane

```
┌ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐
   ＋  무기 올려 강화 시작
└ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘   → /inventory?slot=weapon
```

| grow (등급 사용처) | insaengganghwa 치환 |
|--------------------|----------------------|
| `GRADE_BORDER_CLASS` (스프라이트 박스 테두리) | 초월 단계 테두리/오라 (`transcend_level` 0~10) |

### 3.3 장비 상세 시트 (EquipmentDetailSheet — 초월 실행 위치)

grow 시트 그대로(중앙 모달, swipe-down, a11y). 섹션 순서 동일, 등급→초월 치환:

```
┌── 장비 상세 ──────────────────┐
│      ✦T3 · 무기               │  grow: 등급라벨·슬롯 → 초월·슬롯
│      룬소드  +47              │  이름만
│      🔒 잠금  ✓ 장착          │
│ ┌ [스프라이트] ⚔️ 전투력      │
│ │  초월테두리   612,400 (+...) │  P(L)×(1+초월%), BALANCE §3.2
│ └─────────────────────────┐   │
│ (미장착 시) 현재장착 vs 이장비 │  전투력 diff 비교
│ ── 초월 (신규) ──             │
│  ✦ T3 / 10                    │
│  다음: T4  제물 룬소드 ×4      │  BALANCE §2.1 (선형 1→10)
│  보유 제물 7 (같은 아이템)     │  강화/초월레벨 무관
│  [초월하기] (2탭 확정·즉시)    │  무RNG, T10이면 MAX 비활성
│ ── 액션 ──                    │
│  [⚒️ +47 → +48 강화]          │  slotBusy면 교체 picker
│  [장착/해제]  [잠금/해제]     │
│  [♻️ 분해 — 💎 2]             │  고정 2다이아·2탭
│  [닫기]                       │
└───────────────────────────────┘
```

- **헤더**: grow `GRADE_LABEL·SLOT` → `✦T{초월}·슬롯`. 이름 등급 접두 제거
- **전투력**: `computeCombatPower(base,grade,enh)` → `P(enhanceLevel) × (1 + transcendBonus[T])` (BALANCE §3.1/3.2). 모든 아이템 base 동일(차등 없음, GDD §3.1)
- **초월 섹션(신규)**: 현재 `✦T{n}/10` · 다음 단계 필요 제물 수(BALANCE §2.1 선형 1→10) · 보유 제물 수(같은 catalog_item, 강화/초월 무관, 미장착·비강화중) · `[초월하기]`(2탭 확정, 즉시·무RNG, 제물 부족 시 비활성 / T10이면 `MAX`). 제물 영구 소모(SCHEMA §4)
- **강화 버튼**: grow 그대로(`+N→+N+1 강화`→startEnhance→/강화). slotBusy면 **교체 강화 picker**(slotRunningJobs, 2탭 확정) — picker 내 등급→초월 치환. `isRisky(≥15)`·`MasokWarningModal`·`RISK_ACK_KEY` **전면 삭제**(경고 없음). +100↑ 등록 시 제물 1 필요 표기
- **분해**: grow 2탭 확정 그대로. `DIAMOND_PER_DISENCHANT_BY_GRADE[grade]` → **고정 2다이아**(BALANCE §4.4). `canDisenchant = !장착 && !잠금`
- 장착/해제·잠금/해제·닫기·swipe-down: grow 동일

- grow의 **slot × grade 매트릭스(180) → slot × 카탈로그 아이템**(grade 행/라벨/색/정렬 **제거**)

- **합산 강화 합** = Σ max enhance(전 카탈로그) — 랭킹 합산·총 전투력 도감 보너스 소스(BALANCE §3.2/3.3)
- **도감 보상(claim/claimAll/rewardByGrade) 제거**: grow의 등급별 보석 claim 폐기. 도감의 가치는 **전투력 도감 보너스(×(1+도감강화합×0.005))**로 이미 반영 — 별도 수령 보상 없음(순수 수집 + 전투력 보너스)
- 상세 모달(openEntry): 아이템 정보 — 등급 라벨 제거, 최고 강화/획득 여부, **그 아이템 강화 Top10**(🥇🥈🥉/#순위 · 닉네임→`/u/{nickname}` · `+레벨`, 본인 행 하이라이트, 동률은 먼저 달성 순) + **내 순위**(미획득/미기록=안내). 1위=**챔피언** 표식(👑). 빈 랭킹="아직 기록 없음". 소스 SCHEMA §2.3·BALANCE §3.3
- **챔피언 스프라이트**: 어떤 아이템의 챔피언 유저는 자신의 그 카탈로그 아이템 전 인스턴스가 인벤토리·강화소·자랑(BoastModal)·공개프로필(`/u`)·OG에서 `TranscendSprite isChampion` 변형으로 렌더. 페이지당 챔피언 catalog_item 집합 1쿼리 일괄 조회(N+1 금지, CLAUDE §11.4)
