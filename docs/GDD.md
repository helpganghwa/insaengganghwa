# 인생강화 (insaengganghwa) — Game Design Document (v0)

> **한 줄 컨셉**: "+99 떴다 자랑하기" — 시간기반 idle + 한국식 RPG 강화 하이브리드. 강화 1회에 수 초~수 시간이 걸리고, 그 사이 게임을 떠나 있어도 타이머는 진행된다. 돌아와 결과를 확인하고 장비(또는 세트)를 카톡에 자랑한다.

---

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| 장르 | 시간기반 idle + 한국식 RPG 강화 + 플레이어 호스팅 co-op 레이드 |
| 플랫폼 | 모바일 웹 (PC 호환, 반응형) |
| 타겟 유저 | 한국 모바일 유저, 20~40대, 카카오톡 일상 사용자 |
| 목표 동접 | 100~1000명 |
| 운영 기간 | 5년 이상 |
| 수익 모델 | F2P + IAP (포트원 연동, 카카오페이/토스/카드) |
| 비주얼 | 픽셀아트 (레트로 스타일) |
| 인증 | Kakao OAuth 단독 |
| 본인인증 | KMC/PASS (IAP 사용자 대상 의무) |
| 핵심 동기 | 강화 성공 → 자랑 → 공유 → 신규 유입 |

### MVP 스코프 (1인 개발 현실 반영)

- **Option A — 권장 MVP**: 강화 + 보급 + 공유 + 본인인증 + IAP
- **Option B — 풀 컨텐츠 MVP**: 위 + 레이드 + 어드민

본 문서는 **Option B 기준**. 출시 일정/예산이 빠듯하면 Option A로 축소하고 레이드를 출시 후로 미루는 것을 권장. 스킬트리/영구 패시브는 **미도입**(설계 결정 — 시간 비례 강화 + 천장 없음 + 자동 초월로 메타가 충분히 복잡, 핵심 편의는 기본 기능으로 흡수).

## 2. 핵심 루프 (시간기반 idle)

```
[로그인/접속]
   ↓
[진행 중인 강화 결과 확인] ──→ 성공/실패(유지)/하락 모달 (오프라인 동안 끝난 것)
   ↓                                         ↓
[일일 보급 상자 수령] ──→ [보급 열기]    성공 → 자랑 충동 (장비/세트)
   ↓                       ↓                ↓
[보석 수급]           신규 장비 획득   [카톡 공유] ──→ 신규 유저 유입
 (보급 상자 드롭/                ↓
  레이드/미션)
   ↓                  [다음 강화 큐 등록]
[레이드 도전]              ↓ (수 초~수 시간)
   ↓                  [게임 떠나도 진행]
[랭킹/보상]                ↓
   ↓                  [완료 알림] ──→ 재접속 유도
[다음 사이클 — 보석 단축 / 자랑 / 가챠]
        ↑
[보석으로 즉시 단축] ── IAP
```

### 세션 길이 목표 (idle 패턴 반영)
- **짧은 체크인 세션 (30초~2분)**: 강화 결과 확인 + 다음 강화 큐 등록 + 일일 보급 — 하루 3~5회 반복
- **긴 세션 (10~20분)**: 레이드 도전 + 인벤토리 정리 + 자랑 공유 + 가챠
- **재접속 트리거**: 강화 완료 푸시/이메일, 보스 등장, 친구 공유 알림

### idle 설계 원칙
1. **떠나도 보상이 진행된다** — 강화 타이머는 서버 시계 기준, 클라이언트 미접속 무관
2. **돌아오면 명확한 차이** — "+38 → +39 성공! (3시간 24분 전)" 같은 회상 가능한 결과
3. **단축이 유료의 핵심 동기** — 보석으로 단위 단축, 1회당 만족감 큰 단위로 떨어뜨림
4. **시간 압박 없음** — 강화 큐는 만료되지 않음, 결과는 영구히 대기

## 3. 게임 시스템

### 3.1 장비 시스템

#### 슬롯
- **무기 / 방어구 / 장신구** — 3슬롯 고정
- 4+ 부위 미도입 — 3슬롯 의도적 고정 (세트 단위 자랑 + 모든 부위를 고르게 키우는 동기)

#### 등급 없음 · 베이스 없음
- 장비에 **등급/희소성/부가스탯 체계 없음**. 장비의 강함은 오직 **강화 레벨 + 초월 레벨**에서 나옴
- "베이스(검·활·갑옷…)"는 **슬롯 구분 용도로만** 존재 — 어떤 카탈로그 아이템이 무기/방어구/장신구 중 어느 슬롯에 들어가는지만 결정. 그 외 메커니즘 의미 없음 (전투력·강화 시간 전부 무관)
- **카탈로그 아이템** = 수집·초월의 식별 단위. 각 아이템은 한 슬롯에 속하고, 아이템 간 전투력/시간 차등은 전혀 없음 (외관 + 도감 + 초월 동일성 판정용)

#### 강함의 두 축
1. **강화 (+level)** — 시간기반 강화로 상승. 소요 시간은 **강화 레벨만**으로 결정 (모든 장비 동일 곡선). +51까지 안전 / +52~ 하락. 상세 §3.2
2. **초월 (transcend level)** — 같은 카탈로그 **중복**(박스)이 누적되면 **자동** 상승. 레벨업마다 필요 중복 수 증가, **상한 없음**(무한). 상세 §3.3

#### 전투력
- **개별 장비 전투력** = (강화 레벨 기반 전투력) × (1 + 초월 % 배수) — 정확 수식·계수는 `BALANCE.md` 단일 진실 원천
- **총 전투력** = 보유한 **모든 카탈로그 아이템(중복 제외)의 개별 전투력 합** — 현재 보유 상태 기준
  - **착용 무관**: 착용은 외형(프로필·자랑) 전용 — 전투력에 가중치 없음. 외형을 바꿔도 전투력 불변 → 수집·외형 다양성 장려
  - **카탈로그당 1레코드**: 같은 카탈로그는 1행(중복은 초월 진행도로 흡수). 강화 하락 시 즉시 반영
- 모든 수치는 `lib/game/balance.ts`와 1:1 (본 문서는 모델 설명)

#### 카탈로그
- 카탈로그는 **슬롯별 분류, 종수 가변 — 지속 추가 예정**(§10). 초기 예시: 검/도끼/완드/활/방패/투구/갑옷/장갑/부츠/반지/목걸이/… 각 아이템은 한 슬롯 소속
- 아이템 간 전투력/강화시간 차등 없음 — 카탈로그는 외관 다양성 + 도감 수집 + 초월 동일성 판정 단위
- DB 저장: `(catalog_item_id, enhance_level, transcend_level)` — 등급/베이스스탯/옵션 컬럼 없음

### 3.2 강화 시스템 (시간기반)

게임의 시그니처. 강화 1회 = 시간 소요. 게임을 떠나 있어도 **서버 시계**로 진행된다.

#### 기본 규칙
- 강화 시도 **무료** (강화석·재화·제물 비용 없음). 비용은 **시간**뿐
- 소요 시간 = **강화 레벨만**으로 결정 (카탈로그/슬롯 무관, 모든 장비 동일 곡선). 곡선·수치 = `BALANCE.md`
- **시간 비례 effective rate** — 성공률 = base × (경과 시간 / 총 시간). 끝까지 기다리면 공시 base 도달, 일찍 시도하면 낮은 확률 도박
- 모든 RNG는 **완료 시점 서버**에서만 (CLAUDE §3.1) — 클라이언트는 결과 애니메이션만

#### 위험 구간 / 실패 결과
| 구간 | 실패 시 결과 |
|------|--------------|
| **+0 ~ +51** | **유지** (안전 — 하락 없음) |
| **+52 ~** | **하락** 가능 (단계 하락) |

- **파괴 없음** — 장비가 소멸하는 개념은 도입하지 않음. 최악의 결과는 단계 하락
- **경고 모달 없음** — 위험 구간 진입 시 별도 동의/경고 UX 없음
- 성공률 곡선·하락 폭(몇 단계)·구간 세부 = `BALANCE.md` 단일 진실 원천

#### 동시 진행 / 운영
- 부위당 2 lane × 3 부위 = 최대 **6개 동시 강화**. 빈 lane 자동 할당, 둘 다 차면 `SLOT_BUSY`
- 진행 중 lane **취소** 가능 (강화 무료라 환불 부담 없음) — 슬롯 즉시 해제
- **자동 재등록** — 성공 시 다음 단계 자동 큐잉 (기본 동작). 실패/하락 시 중단, 결과 모달
- **보석으로 시간 단축** — 보석 차감 + `complete_at` 단축. 환산률은 등록 시점 값 영구 유지 (소급 금지). 단위·환산률 = `BALANCE.md`
- 강화 큐는 만료 없음 — 결과 영구 대기 (idle 원칙, §2)

#### 라이프사이클 (트랜잭션)
(A) 큐 등록 → (B) 완료(서버 시계 검증 → RNG → 결과 적용) → (C) 보석 단축 → (D) 취소 / (D+A) 슬롯 교체. (B)는 lazy(접속 조회 시) + cron(미해결 자동 처리)로 멱등 처리. 상세 = `SCHEMA.md` / CLAUDE §6.

### 3.3 초월 시스템

장비의 두 번째 강함 축. 시간(강화)이 아닌 **수집(같은 카탈로그 중복)** 으로 키운다 — 두 축의 역할을 명확히 분리. **수동 조작 없이 자동 진행.**

#### 진행 방식 (자동)
- 보급 상자로 **이미 보유한 카탈로그**가 또 나오면 그 카탈로그의 **초월 진행도**에 +1 누적
- 진행도가 다음 단계 임계에 도달하면 **자동으로 초월 레벨 +1** (한 박스로 여러 단계 동시 가능)
- 유저 선택·확정 UI 없음 — 박스를 여는 것 자체가 곧 초월 진행 (수동 초월/제물 선택 폐기)

#### 레벨업 규칙
- T단계 달성에 같은 카탈로그 중복 **T개** 필요 (선형 증가, `BALANCE.md` 단일 진실 원천)
- **상한 없음** — 무한 진행. 강화처럼 끝없는 엔드게임 축(랜덤 150종 희석이 천장 역할)
- 대상 카탈로그의 **강화 레벨은 초월과 독립** (두 축 분리)

#### 전투력 작용
- 초월 레벨은 **강화 기반 전투력에 % 배수**로 작용 — `개별 장비 전투력 = (강화 기반 전투력) × (1 + 초월 % 배수)`
- 고초월일수록 가치 가속. 정확한 % 계수는 `BALANCE.md`

#### 처리 방식
- **즉시·무RNG** — 박스 열기 트랜잭션 내에서 진행도 누적 + 임계 도달분 자동 레벨업 + 감사 로그 (부분 실패 금지, CLAUDE §3.3)

### 3.4 보급 시스템

장비를 얻는 유일한 경로. 등급이 없으므로 "무슨 등급이 떴나"가 아니라 **어떤 카탈로그 아이템 / 신규냐 중복이냐**가 결과의 의미.

#### 보급 상자 (3종, 슬롯별 분리)
- **무기 보급 상자** → 무기 슬롯 카탈로그 아이템 1개
- **방어구 보급 상자** → 방어구 슬롯 카탈로그 아이템 1개
- **장신구 보급 상자** → 장신구 슬롯 카탈로그 아이템 1개
- 미보유 카탈로그면 **+0 · 초월 0**으로 획득(도감 해금), 보유 중이면 초월 진행도에 누적
- 명칭은 "보급 상자" — 티켓/권 아님

#### 확률 / 천장
- **천장 없음** — 순수 확률. 분쟁 시 신뢰는 정직한 확률 공시로만
- 등급 없음 → 슬롯 박스 내 아이템은 **균등 랜덤** (해당 슬롯 카탈로그 아이템 전부 동일 확률). 공시 표는 BALANCE.md
- 확률 공시 의무 (게임산업법 §33) — 게임 내 1탭 + `/probability` 페이지 동일, 박스별 아이템 확률 전체 표기

#### 중복 처리
- 이미 가진 카탈로그가 또 나오면 **초월 진행도로 자동 누적**(§3.3) — 인벤토리에 중복 개체를 쌓지 않음
- 즉 중복이 곧 초월 성장 — 잉여 정리/분해 같은 관리 행위 자체가 없음
- 도감은 카탈로그 최초 획득 시 활성화 + 해당 카탈로그 최고 강화 레벨 기록

#### 결과 연출
- 등급 이펙트 없음 → 연출 축은 **카탈로그 아이템 등장 + 신규/초월 진행 배지**
- 신규(도감 첫 해금)는 강조 연출, 중복은 "초월 진행 +1"(자동 초월 발동 시 강조) 맥락으로 표기

#### 획득 경로 (개요)
- 일일 무료 / 레이드 보상 / IAP 등 — 구체 수량·한도는 경제·IAP·일일·레이드 섹션에서 정의(보상형 광고 v1 미도입, §3.7)
- 무료 수급은 3슬롯을 고르게 키우도록 슬롯 박스를 균형 배분 (상세 후속 섹션)

### 3.5 레이드 (플레이어 호스팅 co-op)

친구를 카톡으로 불러 함께 보스를 두들기는 소규모 협동 컨텐츠.

#### 개설 / 참여
- 호스트가 **300 다이아**로 개설 (즉시 차감, 환불 없음 — RAID_OPEN_COST_DIAMOND)
- 보스 **5종** 중 선택 — 난이도 동일, 스토리/이미지만 차이. 개설 시 보스 판타지 스토리 팝업
- 개설 시 **공유링크** 생성 → 카톡방 공유 → 링크로 참여(무료). 한 레이드 **최대 10명**(호스트 포함)
- 유저당 동시 진행 레이드 **3개** · **일일 5회**(KST) (호스팅+참여 합산, 강화 슬롯과 같은 동시성 개념)

#### 시간 / 공격
- 개설 후 **6시간**(서버 시계)만 공격 가능
- 참여자당 **기본 10회** 공격 + 다이아로 **추가 공격** 구매 가능 (n번째 점진 증가, BALANCE §5.5)
- **미스 없음** — 모든 공격 명중
- **크리티컬**: 발생 시 데미지 **×1.5** (크리 확률 = `BALANCE.md`)
- 데미지 **±30% 분산** · 데미지 캡 없음
- 데미지는 플레이어 **전투력**(§3.1) 기반 — 정확 수식 = `BALANCE.md`

#### 무한 페이즈
- phase1 HP = 랜덤 8,000~12,000, 페이즈마다 ×1.5 (1.5의 n승)
- **죽이는 개념 없음** — 누적 데미지로 페이즈를 깎고, **6시간 만료로만 종료**
- 깐 페이즈가 많을수록 보상 규모 ↑
- 종료: 6시간 만료 시 lazy(접속 조회) + cron 일괄 정산 (멱등, CLAUDE §3.4)

#### 보상 (전원 동일 — 기여도 무관)
- **기여도 가중 추첨 없음** — 1회 이상 공격한 참여자 **전원이 같은 보상** (기본 참가 보상 없음)
- **페이즈 돌파마다 보급 상자 1개**(전원 동일 적용): 슬롯 무작위(무기·방어구·장신구 각 1/3). 다이아 드롭 없음 — 레이드는 초월용 박스 *보조* 수급처(BALANCE §5.4 시뮬 근거)
- 정확 수치·확률 = `BALANCE.md` §5 단일 진실 원천

#### 보스 5종 (난이도 동일)
- 슬라임킹 · 오크족장 · 돌골렘 · 드래곤(서양식) · 타락천사
- 정적 픽셀 스프라이트 우선(Pixellab), 애니메이션은 수동 후속
- 각 보스 ~150자 판타지 스토리(로어 톤 통일) — 참여/개설 팝업 노출

### 3.6 공유 시스템 (KakaoTalk)

바이럴 핵심 루프 — 강화/초월 성과를 카톡으로 자랑 → 신규 유입.

#### 자랑 단위 (2종)
- **장비 단위** — 장비 1개 상세: 외관(카탈로그 아이템) + 강화 레벨 + 초월 레벨 + 해당 장비 전투력
- **장비 전체** — 무기/방어구/장신구 3슬롯 세트 + 내 프로필(닉네임 · 총 전투력 · 대표 칭호 등)
- 등급 표기 없음 — 자랑의 축은 **강화/초월 레벨 + 전투력**

#### 트리거 (자랑 충동 시점)
- 강화 **+30** 도달
- 강화 **+50** 도달
- 강화 **+99** 도달 (사이클 0 완주 — 큰 성취)
- **첫 초월** (어떤 장비든 초월 레벨 1 최초 달성)
- **고초월 도달** (초월 마일스톤 — 수치는 운영 정책)
- 트리거는 자랑 **유도**(강제 아님). 사용자가 장비 단위/전체 중 선택해 공유

#### 동적 OG 이미지
- Next.js `ImageResponse`로 실시간 생성
- 장비 단위: 해당 카탈로그 아이템 픽셀아트 + 강화/초월/전투력
- 장비 전체: 3슬롯 픽셀아트 + 프로필 요약(닉네임 · 총 전투력 · 대표 칭호)
- 캐싱: 스냅샷 hash 기반 URL → CDN 캐시 (값 바뀌면 새 hash)

#### 구경 모드 (가입 마찰 완화)
- 공유 링크 클릭자가 **비로그인**으로 자랑 콘텐츠 열람 가능
- "나도 강화하기" 클릭 시에만 가입 요구
- OG 페이지에 사회적 증거 표기 가능 (예: 보유자 수 등)

#### 초대 전환 보상
- **공유 링크 → 가입 전환** 시 공유자에게 💎500 + 보급 상자 15개(무기·방어구·장신구 각 5개) — 수치 `BALANCE.md` §6.3

#### 추적
- 공유 링크: `{도메인}/s/{share_code}`
- 공유/클릭/가입 전환 이벤트 테이블 + PostHog 펀널. 상세 = `SCHEMA.md`

### 3.7 광고 보상 — v1 미도입

모바일 웹의 보상형 광고는 SDK 측 서버 검증(SSV) 인프라가 약해 치트 방어가
사실상 불가능. 1인 운영 부담과 한국 모바일 웹 사용자 환경을 고려해 v1에서는
**보상형 광고를 도입하지 않는다**. 잃는 보상은 일일 보급(💎300 + 보급권
3종 5장씩) · 초대 보상 · 레이드 · 강화/초월 진행 등 다른 경로로 충분히 보강된다.

**v2 검토 트리거**: PWA → 네이티브 wrapper(Capacitor 등) 전환 시 AdMob Native
SDK + Google SSV 통합 가능. 그때 §6.4 BALANCE와 함께 재도입 검토.

> 광고 노출(배너/인터스티셜) 자체는 향후 도입 가능하나, **보상은 결합하지 않음**.
> 상점의 광고 제거 IAP(WIREFRAMES §8)는 광고 노출이 추가된 시점에 함께 활성.

**우편함**: 오프라인 완료 강화 결과 · 레이드 정산 · 비동기 보상 적재 (상세 §3.10 / `SCHEMA.md`)

## 4. 사용자 여정 (Day 1)

```
1. 카카오 로그인 (한 번 탭)
   ↓
2. 닉네임 설정 (캐릭터 없음 — 인벤토리만)
   ↓
3. 첫 세트 획득 — 무기/방어구/장신구 보급 상자 각 1개 열기
   → 각 슬롯 카탈로그 아이템 1개씩 (+0 · 초월0) 강제 획득
   ↓
4. 즉시 강화 체험 — 강화 +5까지 즉시 진행 (튜토리얼 동안만 시간 0)
   ↓
5. 강화 큐 개념 학습 — +6 시도 시 "○분 후 완료" 타이머 첫 등장
   → 튜토리얼 보석 5개 무료 지급 → 보석 즉시 단축 체험
   ↓
6. 초월 체험 — 시작 장비 중 1개의 중복 아이템 지급
   → 중복이 자동으로 초월 진행 → 첫 초월(+1) 발동 → 개별/총 전투력 % 상승 확인
   ↓
7. 메인 화면 — 3슬롯 강화 큐를 모두 등록하도록 안내
   ("자고 일어나면 +N 도달")
```

- **3단계**: 등급 없음 — "영웅 세트"가 아니라 슬롯별 보급 상자 열기으로 1세트. 모든 아이템 성능 동일(§3.1)
- **5단계**: 시간 비례 effective rate도 이 시점에 노출 가능 (일찍 시도 = 낮은 확률 도박)
- **6단계**: 두 번째 강함 축(초월) 학습. 중복 = 자동 성장 재료임을 Day 1에 각인 (보급 중복의 가치 이해). 초월 단계별 필요 중복 수는 `BALANCE.md`
- **보스/레이드 없음**: 레이드는 플레이어 호스팅 co-op이라 Day-1 튜토리얼에서 제외 — 메인 진입 후 자연 노출

### Day 1 KPI
- 튜토리얼 완주 ≥ **70%**
- 3슬롯 모두 강화 큐 등록 ≥ **50%** (재접속 동기 핵심 지표)
- D2 재접속 ≥ **30%** (강화 결과 확인)

## 5. 화면 구조

```
[메인 허브]
├── 인벤토리   장비 목록(카탈로그당 1개) + 장착 + 장비 상세(강화 진입·초월 상태 표시)
├── 강화소     선택 장비 + 남은 시간/현재 확률 + 강화 버튼
├── 보급소     무기/방어구/장신구 보급 상자 + 확률 보기 (천장 카운터 없음)
├── 레이드     개설(500다이아) / 카톡 공유 참여 / 세션 공격 (플레이어 호스팅)
├── 랭킹       최고 강화 · 전투력 · 합산 강화 (상시 누적, 시즌제 없음)
├── 상점       IAP — 보석 / 보급 상자 패키지 (광고 제거 IAP는 광고 노출 도입 후 활성)
├── 프로필     칭호 + 도감 + 공유 기록
└── 설정       음향 / 알림 / 계정연동 / 문의

[모달]
├── 강화 결과   성공 / 실패(유지) / 하락 애니메이션 (파괴 없음)
├── 보급 결과   카탈로그 아이템 등장 + 신규/초월 진행 배지 (자동 초월 발동 시 강조)
├── 공유        장비 단위 / 장비 전체 OG 미리보기 + 카카오 공유 버튼
└── 확률 공시   슬롯 박스별 균등 확률 + 강화 확률 일람
```

- **초월**: 별도 진입점·조작 없음 — 보급 상자로 같은 카탈로그 중복이 쌓이면 자동 진행(§3.3). 장비 상세는 현재 초월 레벨·진행도 표시만
- **랭킹**: 시즌 리셋·시즌 보상 없음. 최고 강화 / 전투력 / 합산 강화 3개 기준의 상시 누적 순위(전역)
- **아이템별 챔피언**: 도감에서 카탈로그 아이템 1개당 강화 Top10을 노출(누가 이 장비를 가장 높이 강화했나). 1위 = 그 아이템 **챔피언**(동률 시 그 레벨을 먼저 달성한 유저). 챔피언은 그 카탈로그가 모든 표시처에서 스프라이트 챔피언 변형으로 표시 — "남에게 보이는 영예". 보상 없음(명예만), 결정적(BALANCE §3.3, WIREFRAMES §7.2)
- **보급소**: 천장 카운터 UI 없음 (천장 시스템 자체가 없음). 박스별 균등 확률만 공시

| UI 요소·타일·배경 | 동상 (pixflux) | 자산별 비율 |

- 호출은 `Bearer ${PIXELLAB_API_KEY}`, 응답 `{ image: { base64 } }`, 직접 PNG 저장.
- 동시성 3~4 + 페이싱 0.8s + 429 시 지수 백오프(2→4→8→16→30s). 무료 티어는 버스트 후 강하게 스로틀.
- 재개형 잡 상태 = `scripts/sprite-jobs.json`. 단일 진실 원천 = `lib/game/equipment/catalog.ts`(`art` 필드).

### 프롬프트 작성 원칙
**기본 (모든 아이템)**:
1. **HERO MOTIF 앞배치** — 스토리의 단일 시각 상징을 첫 토큰으로 (`hero motif: ...`).
2. **학습 데이터 강한 단어 회피** — 의도와 다르게 끌고 가는 단어는 동의어로 치환 (예: `slime` → `hardened green resin`).
3. **영역 격리** — 모티프가 묻을 영역과 깨끗할 영역을 양방향으로 강하게 명시 (반대편 영역에 강한 형용사).
4. **palette 3색 명시** — 색 분기를 영역별로 박음.
5. **view 고정** — `side profile` 또는 `front-facing view`.
6. **STYLE + NEG** — 공통 상수 두 개로 마감 (catalog.ts `STYLE`·`NEG`).

**보조 (미세 디테일 케이스)**:
1. 약한 형용사 회피 (`faint`/`subtle` → `clearly visible`/`prominent`/`bold`).
2. 방향성·입체감 명시 (`concave inward`/`recessed`) + 반대편 negative에 박기.
3. 강한 색·재질 단어 비중 축소 (`polished` → `tarnished aged`).
4. 추상 지시는 구체 형상 예시로 (`rune writing` → `X V T angular shapes`).

### 모델 한계 우회 — lore ↔ art 재조정
프롬프트 정교화에도 모델이 표현 못 하는 모티프(예: 128×128에서 미세 7줄 분리)는 **이름·로어를 결과 이미지에 맞춰 다시 쓴다**. catalog의 `nameKo`·`lore`·`art`는 한 셋이고, 셋이 서로 일치하면 어느 쪽이 먼저 정해졌는지는 결과에서 보이지 않는다. 모티프가 모델 한계에 닿으면 art를 억지로 비틀지 않고 lore를 재해석한다.

- 초기 1회 batch 생성 → 1인 큐레이션 → Aseprite 외곽선/팔레트 통일 → 단일 팔레트 락
- 상세 배치 워크플로 = `AUTOMATION.md` (후속 작성)

## 8. 안티치트 & 부정행위 대응

- **모든 RNG는 서버 결정** — 강화는 **큐 완료 시점** 서버 결정(미리 알 수 없음), 보급(슬롯 박스 균등), 레이드 크리. 클라이언트는 결과 애니메이션만
- **시간은 서버 시계만 신뢰** — "이미 완료됐다" 클라이언트 주장 거부. 완료 판정은 항상 `now() >= jobs.complete_at` 서버 비교
- **타임스킵 방지** — 시스템 시계 조작 / DevTools time mocking 무효. 보석 단축 외 시간 가속 불가
- **시드 검증** — 클라이언트 노출 시드는 사후 검증용, 결과 변조 불가
- **레이트 리밋** (Upstash Redis) — 강화 큐 등록 / 보석 단축 / 초월 / 보급 API 호출 빈도 제한
- **자동 초월 무결성** — 박스 열기 시 진행도 누적·자동 레벨업은 서버 트랜잭션 단일 처리(`for update`), 동시성 레이스 차단 (CLAUDE §3.3)
- **이상치 탐지**
  - 동일 IP/디바이스 N계정 자동 플래그
  - 강화 성공률 통계 자동 모니터링 (이상값 알림)
  - 보석 단축 패턴 이상치 (단기 대량 단축 + 즉시 환불)
  - 비정상 데미지 (이론치 초과) 자동 차단
- **감사 로그** — 모든 RNG 결과 + 강화 큐 등록/완료/단축 + 자동 초월 5년 보관, 분쟁 재현 가능
- **결제 어뷰징** — 포트원 webhook 검증, 환불 시 자동 재화 회수 + 환불 보석으로 단축한 강화 결과 동결 검토
- 자가 통계 페이지 없음 — 분쟁은 CS + 감사 로그로 1차 처리
